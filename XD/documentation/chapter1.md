# บทที่ 1 บทนำ

## 1.1 ความเป็นมาและความสำคัญของปัญหา

ภาคการเกษตรเป็นรากฐานสำคัญของเศรษฐกิจไทย โดยมีเกษตรกรไทยมากกว่า 11 ล้านครัวเรือนที่พึ่งพาการเกษตรเป็นอาชีพหลัก อย่างไรก็ตาม เกษตรกรส่วนใหญ่ยังคงเผชิญกับปัญหาและความท้าทายหลายประการ ได้แก่

1. **การตัดสินใจเลือกพืชที่ไม่เหมาะสม** - เกษตรกรมักเลือกปลูกพืชตามประสบการณ์หรือตามกระแส โดยไม่ได้พิจารณาปัจจัยสำคัญอย่างครบถ้วน เช่น สภาพดิน สภาพอากาศ งบประมาณ และความต้องการของตลาด ส่งผลให้ผลผลิตไม่เป็นไปตามที่คาดหวังและขาดทุน

2. **การกำหนดช่วงเวลาปลูกที่ไม่เหมาะสม** - การปลูกในช่วงเวลาที่ไม่เหมาะสมอาจทำให้พืชเจริญเติบโตไม่ดี ได้รับผลกระทบจากภัยธรรมชาติ หรือเก็บเกี่ยวในช่วงที่ราคาตกต่ำ

3. **ความไม่แน่นอนของราคาผลผลิต** - ราคาผลผลิตทางการเกษตรมีความผันผวนสูง เกษตรกรไม่สามารถคาดการณ์ราคาในอนาคตได้ ทำให้วางแผนการผลิตและการเงินได้ยาก

4. **การตัดสินใจเก็บเกี่ยวที่ไม่เหมาะสม** - การเก็บเกี่ยวเร็วหรือช้าเกินไปส่งผลต่อคุณภาพและราคาขาย การตัดสินใจว่าควรเก็บเกี่ยวทันทีหรือรอให้ราคาดีขึ้นเป็นเรื่องที่ท้าทาย

ปัญหาเหล่านี้ส่งผลให้เกษตรกรมีรายได้ไม่แน่นอน ขาดทุน และไม่สามารถพัฒนาคุณภาพชีวิตได้ ในยุคที่เทคโนโลยีดิจิทัลและปัญญาประดิษฐ์ (Artificial Intelligence) ได้รับการพัฒนาอย่างก้าวกระโดด การนำเทคโนโลยีเหล่านี้มาประยุกต์ใช้ในภาคการเกษตรจึงเป็นแนวทางที่มีศักยภาพในการแก้ไขปัญหาดังกล่าว

## 1.2 วัตถุประสงค์ของโครงการ

โครงการนี้มีวัตถุประสงค์หลักดังนี้:

1. **พัฒนาระบบแนะนำพืช (Crop Recommendation System)** ที่สามารถแนะนำพืชที่เหมาะสมกับสภาพพื้นที่ สภาพอากาศ และงบประมาณของเกษตรกร พร้อมทั้งคาดการณ์ผลตอบแทนจากการลงทุน (ROI)

2. **พัฒนาระบบคาดการณ์ช่วงเวลาปลูก (Planting Window Prediction)** ที่สามารถวิเคราะห์และแนะนำช่วงเวลาที่เหมาะสมในการปลูกพืชแต่ละชนิด โดยพิจารณาจากสภาพดิน สภาพอากาศ และปัจจัยตามฤดูกาล

3. **พัฒนาระบบพยากรณ์ราคา (Price Forecasting System)** ที่สามารถคาดการณ์ราคาผลผลิตในอนาคต เพื่อช่วยให้เกษตรกรวางแผนการผลิตและการตลาดได้อย่างมีประสิทธิภาพ

4. **พัฒนาระบบตัดสินใจเก็บเกี่ยว (Harvest Decision System)** ที่สามารถแนะนำเวลาที่เหมาะสมในการเก็บเกี่ยว โดยพิจารณาจากราคาปัจจุบัน การคาดการณ์ราคา สุขภาพของพืช และต้นทุนการเก็บรักษา

5. **บูรณาการระบบทั้ง 4 ส่วนเข้าด้วยกัน** เป็น End-to-End Pipeline ที่สามารถให้คำแนะนำตั้งแต่ก่อนปลูกจนถึงหลังเก็บเกี่ยว

6. **พัฒนาแอปพลิเคชันและแดชบอร์ด** ที่ใช้งานง่าย เข้าถึงได้สะดวก และแสดงผลข้อมูลอย่างชัดเจน

## 1.3 ขอบเขตของโครงการ

### 1.3.1 ขอบเขตด้านพื้นที่

- โครงการนี้มุ่งเน้นพื้นที่การเกษตรในประเทศไทย ครอบคลุม 77 จังหวัด
- ข้อมูลที่ใช้เป็นข้อมูลจากแหล่งต่างๆ ในประเทศไทย

### 1.3.2 ขอบเขตด้านพืช

- ระบบรองรับพืชเศรษฐกิจหลักของไทย จำนวน 46 ชนิด แบ่งเป็น:
  - พืชผัก (ผักใบ ผักผล ผักหัว)
  - พืชไร่ (ข้าว ข้าวโพด มันสำปะหลัง อ้อย)
  - พืชสวน (ผลไม้ต่างๆ)
  - พืชเครื่องเทศ (พริก กระเทียม ขิง)

### 1.3.3 ขอบเขตด้านเทคนิค

- ใช้ Machine Learning Algorithms ได้แก่:
  - XGBoost (Extreme Gradient Boosting)
  - Logistic Regression
  - NSGA-II (Non-dominated Sorting Genetic Algorithm II)
  - Thompson Sampling (Multi-Armed Bandit)
  
- ใช้ Python เป็นภาษาหลักในการพัฒนา
- ใช้ scikit-learn, XGBoost, และ libraries อื่นๆ ที่เกี่ยวข้อง

### 1.3.4 ขอบเขตด้านข้อมูล

- ข้อมูลการเพาะปลูก (Cultivation Data): 6,226 records
- ข้อมูลลักษณะพืช (Crop Characteristics): 46 crops
- ข้อมูลสภาพอากาศ (Weather Data): 56,287 records
- ข้อมูลราคา (Price Data): 2,289,492 records
- ข้อมูลเศรษฐกิจ (Economic Data): 731 records

## 1.4 ประโยชน์ที่คาดว่าจะได้รับ

### 1.4.1 ประโยชน์ต่อเกษตรกร

1. **เพิ่มรายได้** - การเลือกพืชและช่วงเวลาที่เหมาะสมช่วยเพิ่มผลผลิตและลดความเสี่ยง
2. **ลดต้นทุน** - การวางแผนที่ดีช่วยลดการสูญเสียและใช้ทรัพยากรอย่างมีประสิทธิภาพ
3. **ลดความเสี่ยง** - การคาดการณ์ราคาและสภาพอากาศช่วยลดความไม่แน่นอน
4. **ตัดสินใจได้ดีขึ้น** - มีข้อมูลและคำแนะนำที่เชื่อถือได้เป็นฐานในการตัดสินใจ

### 1.4.2 ประโยชน์ต่อภาคการเกษตร

1. **เพิ่มประสิทธิภาพการผลิต** - การใช้ทรัพยากรอย่างเหมาะสมช่วยเพิ่มผลผลิตต่อหน่วยพื้นที่
2. **ความมั่นคงทางอาหาร** - การวางแผนการผลิตที่ดีช่วยให้มีผลผลิตเพียงพอและต่อเนื่อง
3. **การพัฒนาที่ยั่งยืน** - การใช้ข้อมูลและเทคโนโลยีช่วยลดผลกระทบต่อสิ่งแวดล้อม

### 1.4.3 ประโยชน์ทางวิชาการ

1. **องค์ความรู้ใหม่** - การประยุกต์ใช้ AI ในภาคการเกษตรไทย
2. **แนวทางการพัฒนา** - เป็นต้นแบบสำหรับการพัฒนาระบบที่คล้ายคลึงกัน
3. **การป้องกัน Data Leakage** - แนวทางในการสร้างโมเดลที่ถูกต้องและเชื่อถือได้

## 1.5 นิยามศัพท์เฉพาะ

**Machine Learning (ML)** - การเรียนรู้ของเครื่อง เป็นสาขาหนึ่งของปัญญาประดิษฐ์ที่ให้คอมพิวเตอร์สามารถเรียนรู้จากข้อมูลและปรับปรุงประสิทธิภาพได้เองโดยไม่ต้องเขียนโปรแกรมโดยตรง

**Data Leakage** - การรั่วไหลของข้อมูล หมายถึงการที่ข้อมูลที่ไม่ควรมีในขั้นตอนการทำนาย (เช่น ข้อมูลอนาคต) ถูกนำมาใช้ในการฝึกโมเดล ทำให้ผลการประเมินดีเกินจริง

**Feature Engineering** - กระบวนการสร้างและคัดเลือกตัวแปร (features) ที่เหมาะสมจากข้อมูลดิบเพื่อใช้ในการฝึกโมเดล

**ROI (Return on Investment)** - ผลตอบแทนจากการลงทุน คำนวณจาก (กำไร / ต้นทุน) × 100%

**F1-Score** - ค่าวัดประสิทธิภาพของโมเดล Classification ที่พิจารณาทั้ง Precision และ Recall

**R² (R-squared)** - ค่าสัมประสิทธิ์การตัดสินใจ วัดความสามารถของโมเดลในการอธิบายความแปรปรวนของข้อมูล

**Pipeline** - ลำดับขั้นตอนการประมวลผลที่เชื่อมต่อกัน ในที่นี้หมายถึงการเชื่อมโยง Model A → B → C → D

**Thompson Sampling** - อัลกอริทึมสำหรับปัญหา Multi-Armed Bandit ที่ใช้ในการตัดสินใจภายใต้ความไม่แน่นอน

**XGBoost** - Extreme Gradient Boosting อัลกอริทึม Machine Learning ที่มีประสิทธิภาพสูงสำหรับปัญหา Regression และ Classification
