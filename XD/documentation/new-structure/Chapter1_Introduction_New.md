## 1.1 ความเป็นมาและความสำคัญของปัญหา

### 1.1.1 บริบทของภาคเกษตรกรรมไทย

ประเทศไทยมีเกษตรกรมากกว่า 11 ล้านครัวเรือน โดยภาคเกษตรกรรมมีส่วนสำคัญต่อผลิตภัณฑ์มวลรวมภายในประเทศ อย่างไรก็ตาม เกษตรกรรายย่อยยังคงเผชิญกับความท้าทายหลายประการที่ส่งผลกระทบต่อรายได้และความมั่นคงทางอาหารของประเทศ

### 1.1.2 ปัญหาและความท้าทายของเกษตรกร

เกษตรกรไทยต้องเผชิญกับปัญหาที่เชื่อมโยงกันหลายด้าน ประการแรก เกษตรกรต้องตัดสินใจสำคัญด้วยข้อมูลที่ไม่สมบูรณ์ ได้แก่ การเลือกชนิดพืชที่เหมาะสมกับที่ดิน งบประมาณ และสภาวะตลาด การกำหนดช่วงเวลาการเพาะปลูกเพื่อให้ได้ผลผลิตสูงสุดและหลีกเลี่ยงสภาพอากาศที่ไม่เอื้ออำนวย การพยากรณ์ราคาที่จะได้รับในช่วงเก็บเกี่ยว และการกำหนดเวลาการเก็บเกี่ยวเพื่อให้ได้กำไรสูงสุด

ประการที่สอง เกษตรกรรายย่อยเผชิญกับความไม่สมดุลของข้อมูล ในขณะที่บริษัทเกษตรขนาดใหญ่มีเครื่องมือพยากรณ์และข้อมูลตลาดที่ทันสมัย เกษตรกรรายย่อยต้องพึ่งพาความรู้และประสบการณ์แบบดั้งเดิม เครือข่ายไม่เป็นทางการและการบอกต่อ รูปแบบในอดีตที่อาจไม่เหมาะสมอีกต่อไปเนื่องจากการเปลี่ยนแปลงสภาพภูมิอากาศ และการเข้าถึงข้อมูลตลาดแบบเรียลไทม์ที่จำกัด

ประการที่สาม ราคาสินค้าเกษตรในประเทศไทยมีความผันผวนสูง โดยราคาสามารถผันผวน 30-50% ภายในฤดูกาลเดียว เหตุการณ์สภาพอากาศสามารถทำให้เกิดการกระทบกระเทือนด้านอุปทานอย่างกะทันหัน ความต้องการส่งออกมีความไม่แน่นอน และต้นทุนการจัดเก็บและขนส่งเพิ่มความไม่แน่นอนให้กับการตัดสินใจ

ประการที่สี่ การเปลี่ยนแปลงสภาพภูมิอากาศนำมาซึ่งความท้าทายใหม่ ได้แก่ รูปแบบการตกของฝนที่เปลี่ยนแปลง ความถี่ของเหตุการณ์สภาพอากาศรุนแรงที่เพิ่มขึ้น การเปลี่ยนแปลงอุณหภูมิที่ส่งผลต่อวงจรการเจริญเติบโตของพืช และความเสี่ยงจากภัยแล้งและน้ำท่วม

### 1.1.3 ช่องว่างของเทคโนโลยี

แม้จะมีเทคโนโลยีเกษตร (Agricultural Technology) แพร่หลายทั่วโลก แต่การนำไปใช้ในหมู่เกษตรกรรายย่อยของไทยยังคงต่ำเนื่องจากหลายปัจจัย ประการแรก โซลูชันที่มีอยู่มักมีความซับซ้อนและต้องการความเชี่ยวชาญทางเทคนิค ประการที่สอง แพลตฟอร์มเชิงพาณิชย์มีราคาแพงและไม่เหมาะสมกับเกษตรกรรายย่อย ประการที่สาม โซลูชันหลายตัวออกแบบมาสำหรับการเกษตรอุตสาหกรรมขนาดใหญ่ ไม่ใช่เกษตรกรรายย่อย ประการที่สี่ เกษตรกรมีความกังวลเกี่ยวกับความน่าเชื่อถือของคำแนะนำที่ไม่สามารถอธิบายได้ และประการที่ห้า ขาดการบูรณาการกับขั้นตอนการทำงานและแนวปฏิบัติที่มีอยู่

### 1.1.4 โอกาสในการพัฒนาระบบอัจฉริยะ

ระบบ FarmMe แก้ไขความท้าทายเหล่านี้โดยการวางตำแหน่งการสนับสนุนการตัดสินใจทางการเกษตรเป็นส่วนประกอบสำคัญของโครงสร้างพื้นฐานเมืองอัจฉริยะ ข้อมูลการเกษตรสามารถให้ข้อมูลและได้รับประโยชน์จากระบบเมืองอัจฉริยะอื่นๆ เช่น ข้อมูลสภาพอากาศจากเซ็นเซอร์ในเมือง ตัวชี้วัดทางเศรษฐกิจจากระบบการเงิน ข้อมูลการขนส่งเพื่อเพิ่มประสิทธิภาพโลจิสติกส์ และข้อมูลตลาดจากแพลตฟอร์มอีคอมเมิร์ซ

การวางแผนที่ประสานกันในภาคเกษตรช่วยให้สามารถจัดการทรัพยากรน้ำ เพิ่มประสิทธิภาพโครงข่ายไฟฟ้า เพิ่มประสิทธิภาพเครือข่ายการขนส่ง และลดของเสียผ่านการพยากรณ์ความต้องการที่ดีขึ้น นอกจากนี้ ระบบเกษตรอัจฉริยะยังมีส่วนช่วยในการพัฒนาเศรษฐกิจ ความมั่นคงด้านอาหาร เสถียรภาพของราคา ความสามารถในการแข่งขันด้านการส่งออก และการบูรณาการทางเศรษฐกิจระหว่างชนบทและเมือง

## 1.2 วัตถุประสงค์ของโครงการ

### 1.2.1 วัตถุประสงค์หลัก

พัฒนาระบบสนับสนุนการตัดสินใจทางการเกษตรอัจฉริยะที่ใช้เทคโนโลยี Machine Learning เพื่อช่วยเกษตรกรรายย่อยในการตัดสินใจตลอดวงจรการเพาะปลูก ตั้งแต่การเลือกชนิดพืช การกำหนดช่วงเวลาการเพาะปลูก การพยากรณ์ราคาผลผลิต ไปจนถึงการกำหนดเวลาการเก็บเกี่ยว

### 1.2.2 วัตถุประสงค์เฉพาะ

โครงการนี้มุ่งพัฒนาระบบสนับสนุนการตัดสินใจ 4 ด้านที่เชื่อมโยงกันตลอดวงจรการเกษตร โดยเน้นที่ระบบแนะนำการเลือกพืชเป็นหลัก

**วัตถุประสงค์ที่ 1: พัฒนาระบบแนะนำการเลือกพืช (Crop Recommendation System)**

พัฒนาระบบที่สามารถแนะนำชนิดพืชที่เหมาะสมโดยพิจารณาหลายวัตถุประสงค์พร้อมกัน ได้แก่ กำไร ความเสี่ยง และความยั่งยืน ระบบรับข้อมูลนำเข้าประกอบด้วยลักษณะฟาร์ม (ขนาด ประเภทดิน ที่ตั้ง) โปรไฟล์เกษตรกร (งบประมาณ ประสบการณ์) และสภาวะตลาด ผลลัพธ์ที่ได้คือคำแนะนำพืชที่จัดอันดับพร้อมผลตอบแทนจากการลงทุนที่คาดหวัง การประเมินความเสี่ยง และตัวชี้วัดความยั่งยืน โดยใช้เทคโนโลยี NSGA-II (Multi-objective Genetic Algorithm) ร่วมกับ XGBoost

**วัตถุประสงค์ที่ 2: พัฒนาระบบกำหนดช่วงเวลาการเพาะปลูก (Planting Window Classification)**

พัฒนาระบบที่สามารถจำแนกช่วงเวลาการเพาะปลูกที่เหมาะสมสำหรับแต่ละชนิดพืช ระบบรับข้อมูลนำเข้าประกอบด้วยชนิดพืช ที่ตั้ง วันที่ปัจจุบัน สภาพดิน และรูปแบบสภาพอากาศ ผลลัพธ์ที่ได้คือการจำแนกแบบไบนารี (ช่วงเวลาที่เหมาะสม/ไม่เหมาะสม) พร้อมคะแนนความเชื่อมั่น โดยใช้เทคโนโลยี Logistic Regression with Time-aware Features

**วัตถุประสงค์ที่ 3: พัฒนาระบบพยากรณ์ราคาผลผลิต (Price Forecasting)**

พัฒนาระบบที่สามารถพยากรณ์ราคาผลผลิตในช่วงเวลาการเก็บเกี่ยว ระบบรับข้อมูลนำเข้าประกอบด้วยราคาในอดีต ชนิดพืช ที่ตั้ง พยากรณ์อากาศ และตัวชี้วัดทางเศรษฐกิจ ผลลัพธ์ที่ได้คือการพยากรณ์ราคาพร้อมช่วงความเชื่อมั่น โดยใช้เทคโนโลยี XGBoost with Weather and Economic Features เพื่อลด Temporal Bias

**วัตถุประสงค์ที่ 4: พัฒนาระบบตัดสินใจเวลาการเก็บเกี่ยว (Harvest Timing Optimization)**

พัฒนาระบบที่สามารถแนะนำเวลาการเก็บเกี่ยวที่เหมาะสมเพื่อเพิ่มกำไร ระบบรับข้อมูลนำเข้าประกอบด้วยราคาปัจจุบัน การพยากรณ์ราคา สภาพพืช และต้นทุนการจัดเก็บ ผลลัพธ์ที่ได้คือคำแนะนำการดำเนินการ (เก็บเกี่ยวทันที รอ 3 วัน หรือรอ 7 วัน) พร้อมกำไรที่คาดหวัง โดยใช้เทคโนโลยี Thompson Sampling (Multi-Armed Bandit with Bayesian Approach)

### 1.2.3 วัตถุประสงค์ด้านเทคนิค

นอกจากการพัฒนาระบบสนับสนุนการตัดสินใจแล้ว โครงการนี้ยังมุ่งแก้ไขความท้าทายพื้นฐานของ Machine Learning ในด้านการเกษตร

**การป้องกันการรั่วไหลของข้อมูล (Data Leakage Prevention)**: พัฒนาวิธีการตรวจสอบและป้องกันการรั่วไหลของข้อมูลในโมเดล Machine Learning ทางการเกษตร โดยใช้ Temporal Validation และ Feature Auditing อย่างเข้มงวด เพื่อให้โมเดลสามารถทำงานได้จริงในสภาพแวดล้อมการผลิต

**การลดความลำเอียงทางเวลา (Temporal Bias Reduction)**: บูรณาการข้อมูลภายนอก (สภาพอากาศ ตัวชี้วัดเศรษฐกิจ) เพื่อลดการพึ่งพาราคาในอดีต โดยมีเป้าหมายลด Temporal Bias จากร้อยละ 96.79 เหลือร้อยละ 68.09 เพื่อให้โมเดลทำงานได้ดีแม้ในช่วงตลาดผันผวน

**ความสามารถในการอธิบาย (Interpretability)**: พัฒนาระบบอธิบายคำแนะนำที่เข้าใจง่ายสำหรับเกษตรกร โดยใช้ Feature Importance Analysis และ Confidence Scores เพื่อให้เกษตรกรเข้าใจและไว้วางใจคำแนะนำมากขึ้น

**ความพร้อมในการใช้งานจริง (Real-World Deployment)**: สร้างระบบที่แข็งแกร่งด้วย Fallback Mechanisms และ Error Handling ที่สามารถรองรับข้อมูลที่ขาดหายและความล้มเหลวของ API เพื่อให้ระบบพร้อมใช้งานจริงในสภาพแวดล้อมการผลิต

## 1.3 ขอบเขตของโครงการ

โครงการนี้เป็นการพิสูจน์แนวคิด (Proof of Concept) สำหรับการบูรณาการระบบเกษตรอัจฉริยะ โดยมีขอบเขตที่กำหนดไว้อย่างชัดเจนดังนี้

### 1.3.1 ขอบเขตด้านพื้นที่และเวลา

**ขอบเขตด้านเวลา**: ระบบใช้ข้อมูลที่ครอบคลุมระยะเวลา 2 ปี ตั้งแต่วันที่ 1 พฤศจิกายน พ.ศ. 2566 ถึงวันที่ 31 ตุลาคม พ.ศ. 2568 ระยะเวลานี้เพียงพอสำหรับการจับรูปแบบตามฤดูกาลและตรวจสอบแนวทาง อย่างไรก็ตาม ข้อมูลไม่ครอบคลุมแนวโน้มสภาพภูมิอากาศระยะยาว

**ขอบเขตด้านพื้นที่**: ระบบครอบคลุมพื้นที่ประเทศไทยทั้งหมด 77 จังหวัด เนื่องจากสภาพแวดล้อมกฎระเบียบที่เป็นเนื้อเดียวกันและมีข้อมูลที่เข้าถึงได้ อย่างไรก็ตาม ผลลัพธ์อาจไม่สามารถนำไปใช้กับประเทศอื่นได้โดยตรงเนื่องจากความแตกต่างทางภูมิศาสตร์และสภาพภูมิอากาศ

**ขอบเขตด้านพืชผล**: ระบบรองรับพืชเศรษฐกิจหลัก 46 ชนิด ครอบคลุมหลายหมวดหมู่ ได้แก่ ผักใบ ผักผล ผักราก พืชไร่ (ข้าว ข้าวโพด มันสำปะหลัง) ผลไม้ และเครื่องเทศ อย่างไรก็ตาม ระบบไม่รวมพืชเฉพาะทางและพันธุ์รอง

### 1.3.2 ขอบเขตด้านเทคนิค

**แหล่งข้อมูล**: ระบบใช้ข้อมูลสังเคราะห์ (Synthetic Data) ที่สร้างด้วยโมเดลทางสถิติ เนื่องจากเป็นการพิสูจน์แนวคิดและไม่มีการเข้าถึงข้อมูลจริงที่ครอบคลุม ข้อมูลสังเคราะห์ได้รับการตรวจสอบให้มีคุณสมบัติทางสถิติที่สอดคล้องกับรูปแบบการเกษตรที่ทราบ อย่างไรก็ตาม การใช้งานจริงต้องการการบูรณาการข้อมูลจริง

**ทรัพยากรการคำนวณ**: การฝึกโมเดลใช้การเร่งความเร็วด้วย GPU (NVIDIA CUDA) ขณะที่การใช้งานจริงใช้ CPU สำหรับการ Deploy เพื่อสมดุลระหว่างความเร็วในการฝึกและความเป็นจริงในการใช้งาน

**ความซับซ้อนของโมเดล**: ระบบใช้ Classical Machine Learning (XGBoost, Logistic Regression) ร่วมกับ Genetic Algorithms และ Bayesian Methods เนื่องจากความสามารถในการอธิบาย ประสิทธิผลที่พิสูจน์แล้ว และประสิทธิภาพการคำนวณ ระบบไม่ใช้ Deep Learning เนื่องจากข้อมูลไม่เพียงพอและความกังวลด้านการอธิบาย

**ระดับการบูรณาการ**: ปัจจุบันระบบเป็นแบบสแตนด์อโลนพร้อม API Interfaces ในอนาคตสามารถบูรณาการเต็มรูปแบบกับโครงสร้างพื้นฐานข้อมูลเมืองอัจฉริยะได้ อย่างไรก็ตาม การพิสูจน์แนวคิดนี้แสดงความเป็นไปได้ ไม่ใช่การบูรณาการที่สมบูรณ์

### 1.3.3 ข้อจำกัดที่ชัดเจน

เพื่อรักษาความเข้มงวดทางวิทยาศาสตร์ การวิจัยนี้รับทราบข้อจำกัดดังนี้

**ข้อมูลสังเคราะห์**: ข้อมูลการฝึกทั้งหมดสร้างด้วยโมเดลทางสถิติ แม้ว่าจะจับรูปแบบและความสัมพันธ์ที่สมจริงและตรวจสอบกับสถิติการเกษตรที่ทราบ แต่การใช้งานจริงต้องการข้อมูลจริง

**สภาพอากาศแบบพื้นฐาน**: ระบบใช้ตัวแปรอุตุนิยมวิทยาพื้นฐาน (อุณหภูมิ ปริมาณฝน ความชื้น ดัชนีภัยแล้ง) ไม่รวมตัวแปรอื่น เช่น ลม รังสีดวงอาทิตย์ และเซ็นเซอร์ความชื้นในดิน งานในอนาคตควรบูรณาการเซ็นเซอร์ IoT

**เศรษฐศาสตร์แบบพื้นฐาน**: ระบบใช้ตัวชี้วัดเศรษฐกิจระดับมหภาคเท่านั้น (ราคาน้ำมัน ต้นทุนปุ๋ย อัตราเงินเฟ้อ ปริมาณการส่งออก) ไม่รวมพลวัตตลาดระดับจุลภาคและรายละเอียดห่วงโซ่อุปทาน งานในอนาคตควรบูรณาการข้อมูลตลาดแบบเรียลไทม์

**ไม่มีการตรวจสอบในโลกจริง**: ระบบทดสอบด้วยข้อมูลสังเคราะห์เท่านั้น แม้ว่าจะมีวิธีการตรวจสอบที่ครอบคลุมและความเข้มงวดทางสถิติในการประเมิน แต่ไม่ได้วัดผลลัพธ์จากเกษตรกรจริง

**สถานะการพิสูจน์แนวคิด**: ระบบแสดงความเป็นไปได้ ไม่ใช่ความพร้อมในการผลิต แม้ว่าอัลกอริทึมหลักได้รับการตรวจสอบแล้วและสถาปัตยกรรมระบบพิสูจน์แล้ว แต่ต้องการวิศวกรรมเพิ่มเติมสำหรับการใช้งานขนาดใหญ่

## 1.4 ประโยชน์ที่คาดว่าจะได้รับ

**หมายเหตุ**: ประโยชน์ที่ระบุในส่วนนี้เป็นประโยชน์ที่สามารถวัดและตรวจสอบได้จากการทดลองจริง

### 1.4.1 ประโยชน์ด้านประสิทธิภาพของโมเดล

ระบบสามารถสนับสนุนการตัดสินใจทางการเกษตรได้ด้วยประสิทธิภาพที่วัดได้ดังนี้

**ระบบแนะนำการเลือกพืช (Crop Recommendation)**: โมเดล NSGA-II + XGBoost สามารถอธิบายความแปรปรวนของข้อมูลได้ร้อยละ 47 (R² Score = 0.47) และสร้าง Pareto Front ที่แสดงความสมดุลระหว่างกำไร ความเสี่ยง และความยั่งยืน ช่วยให้เกษตรกรสามารถเลือกพืชตามความต้องการของตนเอง

**ระบบกำหนดช่วงเวลาปลูก (Planting Window)**: โมเดล Logistic Regression สามารถจำแนกช่วงเวลาการเพาะปลูกที่เหมาะสมได้ด้วยค่า F1 Score อยู่ในช่วง 0.70-0.75 แสดงให้เห็นถึงความสมดุลระหว่าง Precision และ Recall

**ระบบพยากรณ์ราคา (Price Forecasting)**: โมเดล XGBoost with Weather and Economic Features สามารถพยากรณ์ราคาได้ด้วยค่าความคลาดเคลื่อนเฉลี่ย (MAE) เท่ากับ 13.31 บาท และสามารถลด Temporal Bias จากร้อยละ 96.79 เหลือร้อยละ 68.09 ผ่านการบูรณาการข้อมูลสภาพอากาศและตัวชี้วัดทางเศรษฐกิจ

**ระบบตัดสินใจเก็บเกี่ยว (Harvest Timing)**: โมเดล Thompson Sampling สามารถตัดสินใจเวลาการเก็บเกี่ยวที่เหมาะสมได้ด้วยความแม่นยำร้อยละ 68 โดยใช้แนวทาง Bayesian ในการสมดุลระหว่างการสำรวจและการใช้ประโยชน์

### 1.4.2 ประโยชน์ด้านประสิทธิภาพของระบบ

ระบบมีประสิทธิภาพการทำงานที่วัดได้ดังนี้

**ความเร็วในการประมวลผล**: ระบบสามารถสร้างคำแนะนำการเลือกพืชได้ภายในเวลาไม่เกิน 2 วินาที ทำให้เหมาะสมกับการใช้งานแบบเรียลไทม์

**การเร่งความเร็วด้วย GPU**: กระบวนการสร้างข้อมูลสังเคราะห์ด้วย GPU มีความเร็วเพิ่มขึ้นประมาณ 10 เท่าเมื่อเทียบกับการใช้ CPU เพียงอย่างเดียว

**ขนาดชุดข้อมูล**: ระบบสามารถจัดการข้อมูลขนาดใหญ่ได้ โดยมีข้อมูลทั้งหมด 14 ไฟล์ รวมมากกว่า 2.3 ล้านระเบียน ครอบคลุม 77 จังหวัด และ 46 ชนิดพืช

**สถาปัตยกรรมที่แข็งแกร่ง**: ระบบมี Fallback Mechanisms และ Error Handling ที่สามารถรองรับข้อมูลที่ขาดหายและความล้มเหลวของ API ได้

### 1.4.3 ประโยชน์ด้านวิธีการวิจัย

การวิจัยนี้มีส่วนช่วยในการพัฒนาวิธีการวิจัยด้าน Agricultural Machine Learning ดังนี้

**กรอบการป้องกัน Data Leakage**: พัฒนาวิธีการตรวจสอบและป้องกันการรั่วไหลของข้อมูลอย่างเป็นระบบ ใช้ Temporal Validation และ Feature Auditing อย่างเข้มงวด ทำให้โมเดลสามารถทำงานได้จริงในสภาพแวดล้อมการผลิต

**วิธีการวัด Temporal Bias**: พัฒนาวิธีการวัดและลด Temporal Bias ในโมเดลพยากรณ์ราคา โดยใช้ Feature Importance Analysis เพื่อวัดการพึ่งพาราคาในอดีต

**Multi-Objective Optimization Framework**: นำเสนอกรอบการทำงานสำหรับการเพิ่มประสิทธิภาพหลายวัตถุประสงค์ในการแนะนำการเลือกพืช โดยใช้ NSGA-II ร่วมกับ XGBoost เป็น Fitness Evaluator

**ชุดข้อมูลมาตรฐาน**: สร้างชุดข้อมูลสังเคราะห์ที่สามารถใช้เป็นมาตรฐานสำหรับการทดสอบและเปรียบเทียบโมเดล Agricultural ML โดยมีคุณสมบัติทางสถิติที่สอดคล้องกับรูปแบบการเกษตรที่ทราบ

**กรอบการทดลองที่ทำซ้ำได้**: พัฒนากรอบการทดลองที่สามารถทำซ้ำได้ รวมถึงโปรโตคอลการประเมิน การแบ่งข้อมูล และตัววัดประสิทธิภาพที่เหมาะสมกับปัญหาการเกษตร

## 1.5 โครงสร้างเอกสาร

เอกสารฉบับนี้ประกอบด้วย 5 บทหลัก ดังนี้

**บทที่ 1: บทนำ** นำเสนอความเป็นมาและความสำคัญของปัญหา วัตถุประสงค์ของโครงการ ขอบเขตการศึกษา และประโยชน์ที่คาดว่าจะได้รับจากการพัฒนาระบบ

**บทที่ 2: ทฤษฎีและงานวิจัยที่เกี่ยวข้อง** อธิบายนิยามศัพท์เฉพาะ ทฤษฎีที่เกี่ยวข้องกับการพัฒนาระบบ และงานวิจัยที่เกี่ยวข้องทั้งในและต่างประเทศ

**บทที่ 3: การดำเนินงาน** อธิบายขั้นตอนการดำเนินงานตั้งแต่การเก็บรวบรวมข้อมูล การวิเคราะห์ข้อมูล การสร้างฟีเจอร์ การสร้างและฝึกโมเดล การวัดประสิทธิภาพ และการพัฒนาแอปพลิเคชัน

**บทที่ 4: ผลการทดลอง** นำเสนอผลการทดลองของระบบ โดยเน้นที่ระบบแนะนำการเลือกพืช (Crop Recommendation System) พร้อมการเปรียบเทียบประสิทธิภาพของโมเดลต่างๆ และการสรุปผลตามวัตถุประสงค์

**บทที่ 5: สรุปและอภิปรายผล** สรุปผลการทดลอง อภิปรายผลที่ได้ ข้อจำกัดของการวิจัย ข้อเสนอแนะ และแนวทางการวิจัยในอนาคต

## 1.5 โครงสร้างเอกสาร

เอกสารฉบับนี้ประกอบด้วย 5 บทหลัก โดยแต่ละบทมีเนื้อหาดังนี้

**บทที่ 1: บทนำ** นำเสนอความเป็นมาและความสำคัญของปัญหาที่เกษตรกรไทยเผชิญ วัตถุประสงค์ของโครงการทั้งหมด 4 ด้าน ขอบเขตการศึกษาที่ครอบคลุม 77 จังหวัดและ 46 ชนิดพืช และประโยชน์ที่คาดว่าจะได้รับซึ่งสามารถวัดและตรวจสอบได้จากการทดลองจริง

**บทที่ 2: ทฤษฎีและงานวิจัยที่เกี่ยวข้อง** อธิบายนิยามศัพท์เฉพาะที่ใช้ในการวิจัย ทฤษฎีที่เกี่ยวข้องกับการพัฒนาระบบ ได้แก่ Machine Learning, Multi-Objective Optimization (NSGA-II), Gradient Boosting (XGBoost), Time Series Forecasting, Multi-Armed Bandits (Thompson Sampling) และ Data Leakage Prevention รวมถึงงานวิจัยที่เกี่ยวข้องทั้งในและต่างประเทศ บทนี้จะนำเสนอเฉพาะทฤษฎีและงานวิจัย โดยไม่รวมการอธิบายการนำไปใช้

**บทที่ 3: การดำเนินงาน** อธิบายขั้นตอนการดำเนินงานตามลำดับ ตั้งแต่การเก็บรวบรวมข้อมูล (Data Collection) การวิเคราะห์เชิงสำรวจ (Exploratory Data Analysis) การสร้างฟีเจอร์ (Feature Engineering) การคัดเลือกฟีเจอร์ (Feature Selection) การสร้างและฝึกโมเดลทั้ง 4 โมเดล การแบ่งข้อมูล (Train/Test Split) การวัดประสิทธิภาพ (Evaluation Metrics) การวิเคราะห์ข้อผิดพลาด (Error Analysis) และการพัฒนาแอปพลิเคชัน (Application Development) โดยในแต่ละขั้นตอนจะระบุว่าใช้ทฤษฎีใดจากบทที่ 2

**บทที่ 4: ผลการทดลอง** นำเสนอผลการทดลองของระบบ โดยเน้นที่ระบบแนะนำการเลือกพืช (Crop Recommendation System) ซึ่งเป็นโมเดลหลักของโครงการ พร้อมการเปรียบเทียบประสิทธิภาพของโมเดลต่างๆ ในรูปแบบตาราง การวิเคราะห์ Pareto Front การศึกษากรณีตัวอย่าง และการสรุปผลตามวัตถุประสงค์และขอบเขตที่กำหนดไว้ในบทที่ 1 นอกจากนี้ยังนำเสนอผลการทดลองของโมเดลอื่นๆ (Planting Window, Price Forecasting, Harvest Timing) และแดชบอร์ดของระบบ

**บทที่ 5: สรุปและอภิปรายผล** สรุปผลการทดลองทั้งหมด อภิปรายผลที่ได้โดยอธิบายว่าทำไมโมเดล NSGA-II + XGBoost จึงให้ผลลัพธ์ที่ดีที่สุด พร้อมการอ้างอิงงานวิจัยที่สนับสนุนผลการทดลอง การเปรียบเทียบกับงานวิจัยอื่นที่คล้ายกัน ข้อจำกัดของการวิจัยทั้งด้านข้อมูล ขอบเขต และเทคนิค ข้อเสนอแนะสำหรับการใช้งานจริงและการพัฒนาต่อ และแนวทางการวิจัยในอนาคต

## สรุป

บทนี้ได้นำเสนอบริบท แรงจูงใจ และขอบเขตของระบบสนับสนุนการตัดสินใจทางการเกษตรอัจฉริยะ (FarmMe) โดยวางตำแหน่งงานนี้เป็นการพิสูจน์แนวคิดสำหรับการบูรณาการการสนับสนุนการตัดสินใจทางการเกษตรเข้ากับโครงสร้างพื้นฐานเมืองอัจฉริยะ ระบบนี้แก้ไขความท้าทายที่แท้จริงที่เกษตรกรไทยเผชิญ ได้แก่ ความซับซ้อนในการตัดสินใจ ความไม่สมดุลของข้อมูล ความผันผวนของตลาด และความไม่แน่นอนจากสภาพภูมิอากาศ พร้อมทั้งรับทราบข้อจำกัดของการพิสูจน์แนวคิด

ระบบ FarmMe ประกอบด้วย 4 โมเดล Machine Learning ที่ทำงานร่วมกันตลอดวงจรการเพาะปลูก ได้แก่ ระบบแนะนำการเลือกพืช (NSGA-II + XGBoost) ระบบกำหนดช่วงเวลาปลูก (Logistic Regression) ระบบพยากรณ์ราคา (XGBoost with Weather & Economic Features) และระบบตัดสินใจเก็บเกี่ยว (Thompson Sampling) โดยเน้นที่ระบบแนะนำการเลือกพืชเป็นหลัก

บทต่อไปจะนำเสนอทฤษฎีและงานวิจัยที่เกี่ยวข้องกับการพัฒนาระบบ ซึ่งจะเป็นพื้นฐานสำหรับการดำเนินงานที่จะอธิบายในบทที่ 3
