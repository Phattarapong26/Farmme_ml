## 2.1 นิยามศัพท์เฉพาะ

ส่วนนี้นำเสนอคำนิยามของศัพท์เทคนิคที่ใช้ตลอดทั้งเอกสาร เพื่อให้ผู้อ่านเข้าใจความหมายและบริบทของศัพท์เหล่านี้อย่างชัดเจน

### 2.1.1 ศัพท์ทางด้าน Machine Learning

**Machine Learning (การเรียนรู้ของเครื่อง)** หมายถึง สาขาหนึ่งของปัญญาประดิษฐ์ที่เน้นการพัฒนาอัลกอริทึมและโมเดลทางสถิติที่ช่วยให้ระบบคอมพิวเตอร์สามารถเรียนรู้และปรับปรุงประสิทธิภาพจากประสบการณ์โดยไม่ต้องถูกโปรแกรมอย่างชัดเจน ระบบ Machine Learning เรียนรู้รูปแบบจากข้อมูล (data) และใช้รูปแบบเหล่านั้นในการทำนายหรือตัดสินใจกับข้อมูลใหม่ที่ไม่เคยเห็นมาก่อน

**Supervised Learning (การเรียนรู้แบบมีผู้สอน)** หมายถึง ประเภทหนึ่งของ Machine Learning ที่โมเดลเรียนรู้จากข้อมูลที่มีป้ายกำกับ (labeled data) ซึ่งประกอบด้วยคู่ของข้อมูลนำเข้า (input) และผลลัพธ์ที่ต้องการ (output) โมเดลเรียนรู้ความสัมพันธ์ระหว่าง input และ output เพื่อสามารถทำนาย output สำหรับ input ใหม่ที่ไม่เคยเห็นมาก่อน ตัวอย่างเช่น การทำนายราคาบ้านจากลักษณะของบ้าน หรือการจำแนกอีเมลว่าเป็น spam หรือไม่

**Feature Engineering (การสร้างฟีเจอร์)** หมายถึง กระบวนการสร้าง แปลง หรือคัดเลือกตัวแปร (features) จากข้อมูลดิบเพื่อให้โมเดล Machine Learning สามารถเรียนรู้ได้ดีขึ้น Feature Engineering เป็นขั้นตอนสำคัญที่ต้องใช้ความเข้าใจในโดเมนของปัญหาและความคิดสร้างสรรค์ในการออกแบบ features ที่มีความหมายและมีประโยชน์ต่อการทำนาย ตัวอย่างเช่น การสร้าง feature ใหม่จากวันที่ เช่น วันในสัปดาห์ เดือน หรือฤดูกาล

**Data Leakage (การรั่วไหลของข้อมูล)** หมายถึง สถานการณ์ที่ข้อมูลจากภายนอกชุดข้อมูลการฝึก (training data) รั่วไหลเข้าไปในกระบวนการสร้างโมเดล ทำให้โมเดลมีประสิทธิภาพสูงเกินจริงในระหว่างการฝึกและการทดสอบ แต่ทำงานได้แย่ในสภาพแวดล้อมจริง Data Leakage มักเกิดจากการใช้ข้อมูลที่ไม่ควรมีในเวลาทำนาย เช่น การใช้ข้อมูลอนาคตในการทำนายอดีต หรือการใช้ข้อมูลที่สัมพันธ์โดยตรงกับเป้าหมาย

**Temporal Bias (ความลำเอียงทางเวลา)** หมายถึง ปรากฏการณ์ที่โมเดลพึ่งพาข้อมูลในอดีตมากเกินไป โดยเฉพาะข้อมูลที่ใกล้เคียงกับเวลาปัจจุบัน ทำให้โมเดลไม่สามารถจับรูปแบบที่แท้จริงหรือปัจจัยภายนอกที่มีผลต่อการทำนาย ในบริบทของการพยากรณ์ราคาเกษตร Temporal Bias หมายถึงการที่โมเดลพึ่งพาราคาในอดีตมากเกินไป โดยไม่คำนึงถึงปัจจัยอื่นๆ เช่น สภาพอากาศหรือตัวชี้วัดทางเศรษฐกิจ

**Overfitting (การเรียนรู้มากเกินไป)** หมายถึง สถานการณ์ที่โมเดล Machine Learning เรียนรู้รูปแบบจากข้อมูลการฝึกมากเกินไป รวมถึงสัญญาณรบกวน (noise) ทำให้โมเดลทำงานได้ดีมากกับข้อมูลการฝึก แต่ทำงานได้แย่กับข้อมูลใหม่ที่ไม่เคยเห็นมาก่อน Overfitting เกิดจากโมเดลที่ซับซ้อนเกินไปหรือข้อมูลการฝึกที่ไม่เพียงพอ

**Cross-Validation (การตรวจสอบไขว้)** หมายถึง เทคนิคในการประเมินประสิทธิภาพของโมเดล Machine Learning โดยแบ่งข้อมูลออกเป็นหลายส่วน (folds) และใช้แต่ละส่วนเป็นชุดทดสอบ (test set) ขณะที่ส่วนอื่นๆ เป็นชุดการฝึก (training set) วิธีนี้ช่วยให้ได้การประเมินที่เชื่อถือได้มากขึ้นและลดความเสี่ยงของ overfitting

### 2.1.2 ศัพท์ทางด้านการเกษตร

**Crop Recommendation (การแนะนำการเลือกพืช)** หมายถึง กระบวนการแนะนำชนิดพืชที่เหมาะสมสำหรับการเพาะปลูกในพื้นที่และสภาพแวดล้อมที่กำหนด โดยพิจารณาจากปัจจัยหลายด้าน เช่น ประเภทดิน สภาพภูมิอากาศ ทรัพยากรที่มีอยู่ งบประมาณ ประสบการณ์ของเกษตรกร และสภาวะตลาด การแนะนำการเลือกพืชที่ดีช่วยเพิ่มผลผลิต ลดความเสี่ยง และเพิ่มผลตอบแทนจากการลงทุน

**Planting Window (ช่วงเวลาการเพาะปลูก)** หมายถึง ช่วงเวลาที่เหมาะสมสำหรับการเพาะปลูกพืชแต่ละชนิด ซึ่งขึ้นอยู่กับปัจจัยหลายอย่าง เช่น อุณหภูมิ ปริมาณฝน ความยาวของวัน และวงจรการเจริญเติบโตของพืช การปลูกในช่วงเวลาที่เหมาะสมช่วยให้พืชเจริญเติบโตได้ดี ลดความเสี่ยงจากโรคและแมลง และได้ผลผลิตสูงสุด การปลูกเร็วหรือช้าเกินไปอาจทำให้ผลผลิตลดลง 20-50%

**Harvest Timing (การกำหนดเวลาการเก็บเกี่ยว)** หมายถึง การตัดสินใจว่าควรเก็บเกี่ยวผลผลิตเมื่อใด โดยพิจารณาจากปัจจัยหลายด้าน ได้แก่ ความสุกของพืช ราคาตลาดปัจจุบัน การพยากรณ์ราคาในอนาคต ต้นทุนการจัดเก็บ คุณภาพของผลผลิตที่เสื่อมลงตามเวลา และสภาพอากาศ การกำหนดเวลาการเก็บเกี่ยวที่เหมาะสมช่วยเพิ่มกำไรและลดความเสียหาย

**Yield Prediction (การพยากรณ์ผลผลิต)** หมายถึง การทำนายปริมาณผลผลิตที่จะได้รับจากการเพาะปลูก โดยใช้ข้อมูลหลายด้าน เช่น ประเภทพืช สภาพดิน สภาพอากาศ การจัดการฟาร์ม และข้อมูลในอดีต การพยากรณ์ผลผลิตที่แม่นยำช่วยในการวางแผนการตลาด การจัดการทรัพยากร และการตัดสินใจทางธุรกิจ

**Crop Rotation (การหมุนเวียนพืช)** หมายถึง การปลูกพืชชนิดต่างๆ สลับกันในพื้นที่เดียวกันตามลำดับเวลา เพื่อปรับปรุงคุณภาพดิน ลดโรคและแมลง และเพิ่มผลผลิต การหมุนเวียนพืชเป็นแนวทางการเกษตรที่ยั่งยืนที่ช่วยรักษาความอุดมสมบูรณ์ของดินและลดการใช้สารเคมี

### 2.1.3 ศัพท์ทางด้านการเพิ่มประสิทธิภาพ

**Multi-Objective Optimization (การเพิ่มประสิทธิภาพหลายวัตถุประสงค์)** หมายถึง กระบวนการหาคำตอบที่ดีที่สุดสำหรับปัญหาที่มีวัตถุประสงค์หลายอย่างที่ขัดแย้งกัน เช่น การเพิ่มกำไรและลดความเสี่ยงพร้อมกัน ในปัญหาประเภทนี้ มักไม่มีคำตอบเดียวที่ดีที่สุดสำหรับทุกวัตถุประสงค์ แต่จะมีชุดของคำตอบที่เรียกว่า Pareto Optimal Solutions ซึ่งแต่ละคำตอบมีความสมดุลที่แตกต่างกันระหว่างวัตถุประสงค์ต่างๆ

**Pareto Optimality (ความเหมาะสมแบบพาเรโต)** หมายถึง สถานะของคำตอบที่ไม่สามารถปรับปรุงวัตถุประสงค์หนึ่งได้โดยไม่ทำให้วัตถุประสงค์อื่นแย่ลง คำตอบที่เป็น Pareto Optimal แสดงถึงความสมดุลที่ดีที่สุดระหว่างวัตถุประสงค์ที่ขัดแย้งกัน ในบริบทของการแนะนำการเลือกพืช คำตอบที่เป็น Pareto Optimal คือพืชที่ไม่สามารถเพิ่มกำไรได้โดยไม่เพิ่มความเสี่ยงหรือลดความยั่งยืน

**Pareto Front (แนวหน้าพาเรโต)** หมายถึง ชุดของคำตอบทั้งหมดที่เป็น Pareto Optimal ซึ่งเมื่อแสดงในพื้นที่วัตถุประสงค์ (objective space) จะสร้างเป็นแนวหรือพื้นผิวที่แสดงความสมดุลที่เป็นไปได้ทั้งหมดระหว่างวัตถุประสงค์ต่างๆ Pareto Front ช่วยให้ผู้ตัดสินใจเห็นภาพรวมของทางเลือกที่มีและเลือกคำตอบที่เหมาะสมกับความต้องการของตน

**Genetic Algorithm (อัลกอริทึมเชิงพันธุกรรม)** หมายถึง อัลกอริทึมการเพิ่มประสิทธิภาพที่ได้แรงบันดาลใจจากกระบวนการวิวัฒนาการทางธรรมชาติ โดยใช้หลักการคัดเลือกตามธรรมชาติ (natural selection) การผสมพันธุ์ (crossover) และการกลายพันธุ์ (mutation) เพื่อค้นหาคำตอบที่ดีที่สุด Genetic Algorithm เหมาะสำหรับปัญหาที่ซับซ้อนและมีพื้นที่คำตอบขนาดใหญ่

**Fitness Function (ฟังก์ชันความเหมาะสม)** หมายถึง ฟังก์ชันที่ใช้ประเมินคุณภาพของคำตอบในอัลกอริทึมเชิงพันธุกรรม ฟังก์ชันนี้รับคำตอบเป็น input และคืนค่าที่แสดงถึงความเหมาะสมหรือคุณภาพของคำตอบนั้น คำตอบที่มีค่า fitness สูงกว่ามีโอกาสถูกเลือกเพื่อสร้างคำตอบรุ่นถัดไปมากกว่า

## 2.2 ทฤษฎีที่เกี่ยวข้อง

ส่วนนี้นำเสนอทฤษฎีพื้นฐานที่ใช้ในการพัฒนาระบบ FarmMe โดยเน้นที่ทฤษฎีที่นำไปใช้จริงในโครงการเท่านั้น

### 2.2.1 Machine Learning Fundamentals

Machine Learning เป็นสาขาหนึ่งของปัญญาประดิษฐ์ที่มุ่งเน้นการพัฒนาอัลกอริทึมที่สามารถเรียนรู้รูปแบบจากข้อมูลและใช้รูปแบบเหล่านั้นในการทำนายหรือตัดสินใจ ระบบ Machine Learning แบ่งออกเป็นหลายประเภท แต่ในโครงการนี้เน้นที่ Supervised Learning ซึ่งเป็นประเภทที่โมเดลเรียนรู้จากข้อมูลที่มีป้ายกำกับ

Supervised Learning แบ่งออกเป็นสองประเภทหลักตามลักษณะของผลลัพธ์ที่ต้องการทำนาย ประเภทแรกคือ Regression ซึ่งใช้สำหรับทำนายค่าต่อเนื่อง (continuous values) เช่น ราคา อุณหภูมิ หรือผลผลิต โมเดล Regression เรียนรู้ความสัมพันธ์ระหว่างตัวแปรนำเข้าและค่าผลลัพธ์ที่เป็นตัวเลข ตัวอย่างเช่น การทำนายราคาข้าวจากปัจจัยต่างๆ เช่น ปริมาณฝน อุณหภูมิ และอุปสงค์ในตลาด ประเภทที่สองคือ Classification ซึ่งใช้สำหรับทำนายหมวดหมู่ (categories) เช่น การจำแนกว่าช่วงเวลาหนึ่งเหมาะสมสำหรับการปลูกพืชหรือไม่ หรือการจำแนกประเภทของโรคพืช โมเดล Classification เรียนรู้ขอบเขตการตัดสินใจ (decision boundaries) ที่แยกหมวดหมู่ต่างๆ ออกจากกัน

การประเมินประสิทธิภาพของโมเดล Machine Learning ต้องใช้ตัววัดที่เหมาะสมกับประเภทของปัญหา สำหรับปัญหา Regression ตัววัดที่นิยมใช้ ได้แก่ Mean Absolute Error (MAE) ซึ่งวัดค่าเฉลี่ยของความคลาดเคลื่อนสัมบูรณ์ระหว่างค่าทำนายและค่าจริง Root Mean Squared Error (RMSE) ซึ่งให้น้ำหนักมากกว่ากับความคลาดเคลื่อนที่ใหญ่ และ R² Score (Coefficient of Determination) ซึ่งวัดสัดส่วนของความแปรปรวนในข้อมูลที่โมเดลสามารถอธิบายได้ ค่า R² อยู่ระหว่าง 0 ถึง 1 โดยค่าที่ใกล้ 1 แสดงว่าโมเดลอธิบายข้อมูลได้ดี สำหรับปัญหา Classification ตัววัดที่นิยมใช้ ได้แก่ Accuracy ซึ่งวัดสัดส่วนของการทำนายที่ถูกต้อง Precision ซึ่งวัดสัดส่วนของการทำนายเชิงบวกที่ถูกต้อง Recall ซึ่งวัดสัดส่วนของกรณีเชิงบวกจริงที่ถูกทำนายถูกต้อง และ F1 Score ซึ่งเป็นค่าเฉลี่ยฮาร์มอนิกของ Precision และ Recall ใช้เมื่อต้องการสมดุลระหว่างทั้งสองตัววัด

### 2.2.2 Multi-Objective Optimization และ NSGA-II

การเพิ่มประสิทธิภาพหลายวัตถุประสงค์ (Multi-Objective Optimization) เป็นสาขาหนึ่งของการเพิ่มประสิทธิภาพที่จัดการกับปัญหาที่มีวัตถุประสงค์หลายอย่างที่ต้องการเพิ่มหรือลดพร้อมกัน โดยทั่วไปวัตถุประสงค์เหล่านี้มักขัดแย้งกัน เช่น การเพิ่มกำไรและลดความเสี่ยงพร้อมกัน หรือการเพิ่มคุณภาพและลดต้นทุนพร้อมกัน ในปัญหาประเภทนี้ มักไม่มีคำตอบเดียวที่ดีที่สุดสำหรับทุกวัตถุประสงค์ แต่จะมีชุดของคำตอบที่เรียกว่า Pareto Optimal Solutions

แนวคิดของ Pareto Optimality ถูกนำเสนอโดย Vilfredo Pareto นักเศรษฐศาสตร์ชาวอิตาลี คำตอบหนึ่งเรียกว่าเป็น Pareto Optimal เมื่อไม่สามารถปรับปรุงวัตถุประสงค์หนึ่งได้โดยไม่ทำให้วัตถุประสงค์อื่นแย่ลง ชุดของคำตอบทั้งหมดที่เป็น Pareto Optimal เรียกว่า Pareto Front ซึ่งแสดงถึงความสมดุลที่เป็นไปได้ทั้งหมดระหว่างวัตถุประสงค์ต่างๆ Pareto Front ช่วยให้ผู้ตัดสินใจสามารถเห็นภาพรวมของทางเลือกที่มีและเลือกคำตอบที่เหมาะสมกับความต้องการและข้อจำกัดของตน

NSGA-II (Non-dominated Sorting Genetic Algorithm II) เป็นอัลกอริทึมเชิงพันธุกรรมที่ออกแบบมาสำหรับการเพิ่มประสิทธิภาพหลายวัตถุประสงค์โดยเฉพาะ อัลกอริทึมนี้ถูกพัฒนาโดย Deb และคณะในปี 2002 เป็นการปรับปรุงจาก NSGA เดิม NSGA-II มีจุดเด่นหลายประการ ได้แก่ Fast Non-dominated Sorting ซึ่งลดความซับซ้อนในการคำนวณจาก O(MN³) เหลือ O(MN²) โดยที่ M คือจำนวนวัตถุประสงค์และ N คือขนาดประชากร Crowding Distance ซึ่งใช้รักษาความหลากหลายของคำตอบบน Pareto Front และ Elitism ซึ่งรับประกันว่าคำตอบที่ดีที่สุดจะไม่สูญหายในรุ่นถัดไป

กระบวนการทำงานของ NSGA-II เริ่มจากการสร้างประชากรเริ่มต้นแบบสุ่ม จากนั้นในแต่ละรุ่น (generation) อัลกอริทึมจะสร้างประชากรลูก (offspring population) ผ่านการคัดเลือก (selection) การผสมพันธุ์ (crossover) และการกลายพันธุ์ (mutation) จากนั้นรวมประชากรพ่อแม่และประชากรลูกเข้าด้วยกัน และทำ Fast Non-dominated Sorting เพื่อแบ่งคำตอบออกเป็นชั้น (fronts) ตามระดับการ dominate คำตอบที่ไม่ถูก dominate โดยคำตอบอื่นจะอยู่ใน Front ที่ 1 คำตอบที่ถูก dominate เฉพาะโดยคำตอบใน Front ที่ 1 จะอยู่ใน Front ที่ 2 และต่อไปเรื่อยๆ หลังจากนั้นคำนวณ Crowding Distance สำหรับคำตอบในแต่ละ Front เพื่อวัดความหนาแน่นของคำตอบในพื้นที่วัตถุประสงค์ สุดท้ายเลือกคำตอบที่ดีที่สุดตามลำดับ Front และ Crowding Distance เพื่อสร้างประชากรรุ่นถัดไป กระบวนการนี้ทำซ้ำจนกว่าจะถึงเงื่อนไขหยุด เช่น จำนวนรุ่นสูงสุดหรือคุณภาพของคำตอบที่ยอมรับได้

### 2.2.3 Gradient Boosting และ XGBoost

Gradient Boosting เป็นเทคนิค Ensemble Learning ที่สร้างโมเดลที่แข็งแกร่ง (strong learner) จากการรวมโมเดลที่อ่อนแอ (weak learners) หลายตัวเข้าด้วยกัน โดยทั่วไปใช้ Decision Trees เป็น weak learners แนวคิดหลักของ Gradient Boosting คือการสร้างโมเดลใหม่ที่พยายามแก้ไขข้อผิดพลาดของโมเดลก่อนหน้า โดยใช้ Gradient Descent ในพื้นที่ฟังก์ชัน (function space) แทนที่จะเป็นพื้นที่พารามิเตอร์ (parameter space) เหมือนใน Neural Networks

กระบวนการทำงานของ Gradient Boosting เริ่มจากการสร้างโมเดลเริ่มต้นที่ทำนายค่าคงที่ เช่น ค่าเฉลี่ยของเป้าหมาย จากนั้นในแต่ละรอบ (iteration) คำนวณ residuals ซึ่งเป็นความแตกต่างระหว่างค่าจริงและค่าทำนายปัจจุบัน สร้าง Decision Tree ใหม่เพื่อทำนาย residuals เหล่านี้ และเพิ่ม tree ใหม่เข้าไปในโมเดลโดยคูณด้วย learning rate ซึ่งควบคุมอิทธิพลของแต่ละ tree กระบวนการนี้ทำซ้ำหลายรอบ โดยแต่ละ tree ใหม่พยายามแก้ไขข้อผิดพลาดที่เหลืออยู่จาก trees ก่อนหน้า ผลลัพธ์สุดท้ายคือการรวมของ trees ทั้งหมด ซึ่งสามารถจับรูปแบบที่ซับซ้อนในข้อมูลได้

XGBoost (eXtreme Gradient Boosting) เป็นการพัฒนาต่อยอดจาก Gradient Boosting ที่ออกแบบมาเพื่อประสิทธิภาพและความเร็วสูง อัลกอริทึมนี้ถูกพัฒนาโดย Chen และ Guestrin ในปี 2016 และกลายเป็นหนึ่งในอัลกอริทึมที่นิยมใช้มากที่สุดสำหรับข้อมูลแบบตาราง (tabular data) XGBoost มีจุดเด่นหลายประการที่ทำให้เหนือกว่า Gradient Boosting แบบดั้งเดิม ได้แก่ Regularization ซึ่งเพิ่ม L1 และ L2 regularization terms เข้าไปในฟังก์ชันวัตถุประสงค์เพื่อป้องกัน overfitting การจัดการกับ Missing Values โดยอัตโนมัติผ่านการเรียนรู้ทิศทางที่ดีที่สุดสำหรับค่าที่หายไป Parallel Processing ซึ่งใช้ประโยชน์จาก multi-core processors เพื่อเร่งความเร็วในการสร้าง trees และ Tree Pruning ซึ่งใช้ max_depth parameter เพื่อควบคุมความซับซ้อนของ trees

นอกจากนี้ XGBoost ยังมี Built-in Cross-Validation ที่ช่วยในการปรับแต่ง hyperparameters Early Stopping ที่หยุดการฝึกเมื่อประสิทธิภาพบน validation set ไม่ดีขึ้น และ Feature Importance ที่ช่วยในการเข้าใจว่า features ใดมีผลต่อการทำนายมากที่สุด ด้วยคุณสมบัติเหล่านี้ XGBoost จึงเหมาะสำหรับปัญหาที่หลากหลาย ทั้ง regression และ classification และมักให้ผลลัพธ์ที่ดีกว่าอัลกอริทึมอื่นๆ ในการแข่งขัน Machine Learning เช่น Kaggle

### 2.2.4 Time Series Forecasting

Time Series Forecasting เป็นสาขาหนึ่งของ Machine Learning ที่เน้นการทำนายค่าในอนาคตจากข้อมูลที่เรียงตามลำดับเวลา ข้อมูล Time Series มีลักษณะพิเศษคือมีความสัมพันธ์ทางเวลา (temporal dependency) ซึ่งหมายความว่าค่าในเวลาหนึ่งมักเกี่ยวข้องกับค่าในเวลาก่อนหน้า นอกจากนี้ข้อมูล Time Series มักมีรูปแบบตามฤดูกาล (seasonality) และแนวโน้ม (trend) ที่ต้องคำนึงถึงในการสร้างโมเดล

วิธีการแบบดั้งเดิมสำหรับ Time Series Forecasting ได้แก่ ARIMA (AutoRegressive Integrated Moving Average) ซึ่งเป็นโมเดลทางสถิติที่รวม Autoregression (AR) ซึ่งใช้ค่าในอดีตเพื่อทำนายค่าปัจจุบัน Moving Average (MA) ซึ่งใช้ข้อผิดพลาดในอดีตเพื่อทำนายค่าปัจจุบัน และ Integration (I) ซึ่งทำให้ข้อมูลมีความนิ่ง (stationary) ผ่านการหาผลต่าง (differencing) ARIMA เหมาะสำหรับข้อมูลที่มีความนิ่งและมีรูปแบบเชิงเส้น แต่มีข้อจำกัดในการจับรูปแบบที่ซับซ้อนและไม่เป็นเชิงเส้น

Machine Learning สำหรับ Time Series Forecasting มักใช้ Feature Engineering เพื่อแปลงข้อมูล Time Series ให้อยู่ในรูปแบบที่โมเดล Machine Learning สามารถเรียนรู้ได้ Features ที่นิยมใช้ ได้แก่ Lag Features ซึ่งเป็นค่าในอดีตที่ช่วงเวลาต่างๆ เช่น ราคาเมื่อ 1 วันก่อน 7 วันก่อน หรือ 30 วันก่อน Rolling Statistics ซึ่งเป็นสถิติที่คำนวณจากหน้าต่างเลื่อน (rolling window) เช่น ค่าเฉลี่ย ส่วนเบี่ยงเบนมาตรฐาน ค่าต่ำสุด และค่าสูงสุดในช่วง 7 วันที่ผ่านมา Date Features ซึ่งสกัดจากวันที่ เช่น วันในสัปดาห์ เดือน ไตรมาส หรือฤดูกาล และ Cyclical Encoding ซึ่งแปลง features ที่มีรูปแบบวนซ้ำ เช่น เดือนหรือวันในสัปดาห์ ให้อยู่ในรูปแบบ sine และ cosine เพื่อรักษาความต่อเนื่องของรูปแบบวนซ้ำ

การใช้ Machine Learning กับ Time Series มีข้อควรระวังสำคัญคือต้องป้องกัน Data Leakage โดยเคร่งครัด ซึ่งหมายความว่าต้องไม่ใช้ข้อมูลอนาคตในการทำนายอดีต การแบ่งข้อมูลสำหรับ Time Series ต้องใช้ Temporal Split ซึ่งแบ่งข้อมูลตามเวลา โดยข้อมูลในอดีตเป็น training set และข้อมูลในอนาคตเป็น test set ไม่สามารถใช้ Random Split เหมือนกับปัญหาทั่วไปได้ นอกจากนี้ การสร้าง features ต้องระวังไม่ให้รวมข้อมูลอนาคต เช่น การคำนวณ rolling statistics ต้อง shift ไปอย่างน้อย 1 period เพื่อไม่ให้รวมค่าปัจจุบัน

## 2.2.5 Multi-Armed Bandits และ Thompson Sampling

Multi-Armed Bandit Problem เป็นปัญหาคลาสสิกในทฤษฎีการตัดสินใจที่เกี่ยวข้องกับการสมดุลระหว่างการสำรวจ (exploration) และการใช้ประโยชน์ (exploitation) ปัญหานี้ได้ชื่อมาจากเครื่องสล็อตในคาสิโน (one-armed bandit) ที่มีหลายเครื่อง (multi-armed) โดยแต่ละเครื่องให้รางวัลที่แตกต่างกันตามการแจกแจงความน่าจะเป็นที่ไม่ทราบ เป้าหมายคือเลือกเครื่องที่จะเล่นในแต่ละครั้งเพื่อเพิ่มรางวัลสะสมให้มากที่สุด ความท้าทายคือต้องสมดุลระหว่างการสำรวจเครื่องต่างๆ เพื่อเรียนรู้ว่าเครื่องใดให้รางวัลดีที่สุด และการใช้ประโยชน์จากความรู้ที่มีอยู่โดยเลือกเครื่องที่ดีที่สุดที่รู้จัก

ในบริบทของการตัดสินใจทางการเกษตร Multi-Armed Bandit สามารถนำมาประยุกต์ใช้กับปัญหาการกำหนดเวลาการเก็บเกี่ยวได้ โดยแต่ละ "arm" แทนการตัดสินใจที่แตกต่างกัน เช่น เก็บเกี่ยวทันที รอ 3 วัน หรือรอ 7 วัน รางวัลคือกำไรที่ได้รับจากการตัดสินใจนั้น ซึ่งขึ้นอยู่กับราคาตลาด คุณภาพของผลผลิต และต้นทุนการจัดเก็บ เกษตรกรต้องเรียนรู้ว่าการตัดสินใจใดให้กำไรสูงสุดในสถานการณ์ต่างๆ โดยสมดุลระหว่างการทดลองตัวเลือกใหม่และการใช้ตัวเลือกที่รู้ว่าดี

อัลกอริทึมสำหรับ Multi-Armed Bandit มีหลายประเภท อัลกอริทึมที่เรียบง่ายที่สุดคือ ε-Greedy ซึ่งเลือก arm ที่ดีที่สุดที่รู้จักด้วยความน่าจะเป็น 1-ε และเลือก arm แบบสุ่มด้วยความน่าจะเป็น ε เพื่อสำรวจ อัลกอริทึมนี้ง่ายแต่มีข้อเสียคือเสียเวลาในการสำรวจ arms ที่ชัดเจนว่าไม่ดี อัลกอริทึมที่ซับซ้อนกว่าคือ Upper Confidence Bound (UCB) ซึ่งเลือก arm โดยพิจารณาทั้งค่าเฉลี่ยของรางวัลและความไม่แน่นอน (uncertainty) ของการประมาณ UCB เลือก arm ที่มีค่า upper confidence bound สูงสุด ซึ่งสมดุลระหว่างการใช้ประโยชน์จาก arms ที่ให้รางวัลสูงและการสำรวจ arms ที่มีความไม่แน่นอนสูง

Thompson Sampling เป็นอัลกอริทึม Bayesian สำหรับ Multi-Armed Bandit ที่มีประสิทธิภาพสูงและมีพื้นฐานทางทฤษฎีที่แข็งแกร่ง อัลกอริทึมนี้รักษาการแจกแจงความน่าจะเป็นแบบ posterior สำหรับพารามิเตอร์ของแต่ละ arm ซึ่งแสดงถึงความเชื่อปัจจุบันเกี่ยวกับคุณภาพของ arm นั้น ในแต่ละรอบ Thompson Sampling สุ่มตัวอย่างพารามิเตอร์จาก posterior distribution ของแต่ละ arm และเลือก arm ที่มีค่าตัวอย่างสูงสุด หลังจากสังเกตรางวัล posterior distribution ถูกอัปเดตโดยใช้ Bayes' theorem กระบวนการนี้สมดุลระหว่างการสำรวจและการใช้ประโยชน์โดยอัตโนมัติ โดย arms ที่มีความไม่แน่นอนสูงมีโอกาสถูกเลือกมากกว่าเพื่อลดความไม่แน่นอน ขณะที่ arms ที่รู้ว่าดีก็มีโอกาสถูกเลือกมากเช่นกัน

สำหรับปัญหาที่รางวัลเป็น Bernoulli (สำเร็จหรือล้มเหลว) Thompson Sampling ใช้ Beta distribution เป็น prior และ posterior ซึ่งเป็น conjugate prior ทำให้การอัปเดตทำได้ง่ายและรวดเร็ว สำหรับปัญหาที่รางวัลเป็น Gaussian Thompson Sampling ใช้ Normal-Gamma distribution เป็น prior และ posterior ข้อดีของ Thompson Sampling คือมีพื้นฐานทางทฤษฎีที่แข็งแกร่ง มีการพิสูจน์ว่ามี regret bound ที่เหมาะสม (optimal) สามารถรวมความรู้เบื้องต้น (prior knowledge) เข้าไปได้ และมีประสิทธิภาพสูงในทางปฏิบัติ

### 2.2.6 Data Leakage Prevention

Data Leakage เป็นปัญหาสำคัญใน Machine Learning ที่มักถูกมองข้าม แต่สามารถทำให้ผลการประเมินโมเดลคลาดเคลื่อนอย่างมากและทำให้โมเดลทำงานได้แย่ในสภาพแวดล้อมจริง Data Leakage เกิดขึ้นเมื่อข้อมูลจากภายนอกชุดข้อมูลการฝึกรั่วไหลเข้าไปในกระบวนการสร้างโมเดล ทำให้โมเดลเรียนรู้รูปแบบที่ไม่สามารถใช้ได้ในการทำนายจริง ผลที่ตามมาคือโมเดลมีประสิทธิภาพสูงเกินจริงในระหว่างการฝึกและการทดสอบ แต่ทำงานได้แย่มากเมื่อนำไปใช้จริง

Data Leakage แบ่งออกเป็นหลายประเภท ประเภทแรกคือ Temporal Leakage ซึ่งเกิดขึ้นเมื่อใช้ข้อมูลอนาคตในการทำนายอดีต ปัญหานี้พบบ่อยใน Time Series Forecasting เช่น การใช้ราคาในวันถัดไปเพื่อทำนายราคาวันนี้ หรือการคำนวณ rolling statistics โดยไม่ shift ทำให้รวมค่าปัจจุบันด้วย ประเภทที่สองคือ Target Leakage ซึ่งเกิดขึ้นเมื่อ features มีความสัมพันธ์โดยตรงกับ target หรือถูกสร้างจาก target เช่น การใช้ "จำนวนครั้งที่ผิดนัดชำระ" เพื่อทำนาย "ความเสี่ยงในการผิดนัดชำระ" ประเภทที่สามคือ Train-Test Contamination ซึ่งเกิดขึ้นเมื่อข้อมูลจาก test set มีอิทธิพลต่อการฝึกโมเดล เช่น การ fit scaler หรือ imputer บนข้อมูลทั้งหมดก่อนแบ่ง train-test ทำให้สถิติจาก test set รั่วไหลเข้าไปใน training process

การตรวจจับ Data Leakage สามารถทำได้หลายวิธี วิธีแรกคือการตรวจสอบประสิทธิภาพที่ดีเกินจริง ถ้าโมเดลมี R² > 0.99 หรือ accuracy = 100% บนข้อมูลที่มีสัญญาณรบกวนสูง น่าจะมี leakage วิธีที่สองคือการวิเคราะห์ Feature Importance ถ้า feature ใดมีความสำคัญสูงมากแต่ไม่ควรมีความสัมพันธ์กับ target ตามตรรกะ อาจเป็นสัญญาณของ leakage วิธีที่สามคือการตรวจสอบความสอดคล้องทางเวลา โดยฝึกโมเดลบนข้อมูลช่วงหนึ่งและทดสอบบนข้อมูลช่วงอื่น ถ้าประสิทธิภาพลดลงอย่างมาก อาจมี temporal leakage วิธีที่สี่คือการวิเคราะห์ความสัมพันธ์ระหว่าง features และ target ถ้ามีความสัมพันธ์สูงมาก (|correlation| > 0.95) ควรตรวจสอบว่า feature นั้นถูกสร้างอย่างไร

การป้องกัน Data Leakage ต้องใช้ความระมัดระวังในทุกขั้นตอนของการพัฒนาโมเดล หลักการสำคัญคือต้องจำลองสภาพแวดล้อมการใช้งานจริงอย่างเคร่งครัด สำหรับ Time Series ต้องใช้ Temporal Split เสมอ โดยแบ่งข้อมูลตามเวลา ไม่ใช้ Random Split และต้องแน่ใจว่าทุก feature ใช้เฉพาะข้อมูลที่มีอยู่ในเวลาทำนาย สำหรับ Feature Engineering ต้องเอกสารว่าแต่ละ feature มีข้อมูลเมื่อใด และต้อง shift rolling statistics อย่างน้อย 1 period สำหรับ Data Preprocessing ต้องใช้ sklearn Pipeline เพื่อให้แน่ใจว่า transformations ทั้งหมด (scaling, imputation, encoding) ถูก fit บน training data เท่านั้นและถูก apply กับ test data โดยใช้สถิติจาก training data สำหรับ Cross-Validation ต้องใช้ TimeSeriesSplit สำหรับข้อมูล Time Series และต้องแน่ใจว่า test fold อยู่หลัง train fold เสมอ

การป้องกัน Data Leakage ในระบบ Agricultural ML มีความสำคัญเป็นพิเศษเพราะข้อมูลการเกษตรมักมีโครงสร้างทางเวลาที่ซับซ้อน มีรูปแบบตามฤดูกาล และมีความล่าช้าในการได้รับข้อมูล การใช้ข้อมูลที่ไม่มีในเวลาทำนายจริงจะทำให้โมเดลมีประสิทธิภาพสูงเกินจริงในการทดสอบ แต่ล้มเหลวในการใช้งานจริง ดังนั้นการพัฒนาโมเดล Agricultural ML ต้องมีความเข้มงวดในการป้องกัน Data Leakage และต้องมีการตรวจสอบอย่างละเอียดว่าทุก feature และทุกขั้นตอนของ pipeline ไม่มีการรั่วไหลของข้อมูล

## 2.3 งานวิจัยที่เกี่ยวข้อง

ส่วนนี้นำเสนองานวิจัยที่เกี่ยวข้องกับระบบ FarmMe ทั้งระบบเชิงพาณิชย์และงานวิจัยทางวิชาการ เพื่อแสดงให้เห็นถึงบริบทและช่องว่างการวิจัยที่ระบบนี้พยายามแก้ไข

### 2.3.1 ระบบ Machine Learning ทางการเกษตร

การประยุกต์ใช้ Machine Learning ในภาคเกษตรกรรมได้พัฒนาผ่านหลายยุค เริ่มจากระบบผู้เชี่ยวชาญ (Expert Systems) ในทศวรรษ 1980-1990 ซึ่งใช้กฎที่เข้ารหัสจากความรู้ของผู้เชี่ยวชาญ เช่น COMAX สำหรับการจัดการฝ้าย และ GOSSYM สำหรับการจำลองการเจริญเติบโตของพืช ระบบเหล่านี้มีข้อจำกัดคือต้องการการสร้างความรู้ด้วยมือและยากต่อการบำรุงรักษา ในทศวรรษ 1990-2000 มีการใช้โมเดลทางสถิติ เช่น Linear Regression สำหรับการทำนายผลผลิตและ Logistic Regression สำหรับการจำแนกโรค แต่โมเดลเหล่านี้มีข้อจำกัดในการจับรูปแบบที่ซับซ้อน

ในทศวรรษ 2000-2010 เริ่มมีการใช้ Machine Learning อย่างแพร่หลาย โดยใช้ Support Vector Machines สำหรับการจำแนกพืช Random Forests สำหรับการทำนายผลผลิต และ Neural Networks สำหรับการตรวจจับโรคพืชจากภาพ ปัจจุบัน (ทศวรรษ 2010 เป็นต้นมา) มีการใช้ Deep Learning เช่น Convolutional Neural Networks สำหรับการตรวจสอบพืชจากภาพถ่ายดาวเทียม Recurrent Neural Networks สำหรับการพยากรณ์ Time Series และการบูรณาการข้อมูลจากหลายแหล่ง เช่น ภาพดาวเทียม เซ็นเซอร์ IoT และข้อมูลสภาพอากาศ

### 2.3.2 ระบบแนะนำการเลือกพืช

งานวิจัยด้านระบบแนะนำการเลือกพืชมีหลายแนวทาง แนวทางแบบดั้งเดิมใช้ระบบกฎ (Rule-Based Systems) ที่เข้ารหัสความรู้ของผู้เชี่ยวชาญเป็นกฎ if-then เช่น "ถ้า pH ของดิน > 7 และปริมาณฝน > 1000 มม. แนะนำให้ปลูกข้าว" ระบบเหล่านี้มีข้อจำกัดคือไม่สามารถจัดการกับความไม่แน่นอนและยากต่อการบำรุงรักษา แนวทางอื่นใช้โมเดลทางสถิติ เช่น Linear Regression ที่ทำนายผลผลิตจากปัจจัยต่างๆ แต่มีข้อจำกัดคือสมมติความสัมพันธ์เชิงเส้นและไม่สามารถจับปฏิสัมพันธ์ที่ซับซ้อน

แนวทางสมัยใหม่ใช้ Machine Learning โดยแบ่งออกเป็นหลายประเภท ประเภทแรกคือ Classification-Based Systems ที่ถือว่าการแนะนำพืชเป็นปัญหา multi-class classification โดยใช้อัลกอริทึม เช่น Decision Trees, Random Forest, SVM และ Neural Networks ตัวอย่างเช่น งานวิจัยของ Pudumalar และคณะ (2016) ใช้ Naive Bayes สำหรับการทำนายพืชได้ความแม่นยำ 88% ประเภทที่สองคือ Regression-Based Systems ที่ทำนายผลผลิตสำหรับแต่ละพืชและแนะนำพืชที่ให้ผลผลิตสูงสุด โดยใช้อัลกอริทึม เช่น Linear Regression, Ridge, Lasso และ XGBoost ตัวอย่างเช่น งานวิจัยของ Khosla และคณะ (2010) ใช้ Regression Models สำหรับ Precision Agriculture ประเภทที่สามคือ Ensemble Methods ที่รวมหลายโมเดลเพื่อความแข็งแกร่ง ตัวอย่างเช่น งานวิจัยของ Ramesh และ Vardhan (2015) ใช้ Ensemble ของ Decision Tree, Random Forest และ SVM ได้ความแม่นยำ 92%

อย่างไรก็ตาม ระบบที่มีอยู่มีข้อจำกัดหลายประการ ประการแรก ส่วนใหญ่ใช้ Single-Objective Optimization โดยมุ่งเน้นเฉพาะกำไรหรือผลผลิต ไม่คำนึงถึงความเสี่ยงหรือความยั่งยืน ประการที่สอง มีการจัดการกับข้อจำกัดของเกษตรกร (งบประมาณ ที่ดิน ประสบการณ์) อย่างจำกัด ประการที่สาม โมเดลที่ซับซ้อนขาดความสามารถในการอธิบาย ทำให้เกษตรกรไม่เข้าใจและไม่ไว้วางใจคำแนะนำ

### 2.3.3 ระบบพยากรณ์ราคาเกษตร

การพยากรณ์ราคาสินค้าเกษตรเป็นปัญหาที่ท้าทายเนื่องจากราคามีความผันผวนสูง (ส่วนเบี่ยงเบนมาตรฐาน 15-30%) มีรูปแบบตามฤดูกาล (ราคาลดลงในช่วงเก็บเกี่ยว) และได้รับผลกระทบจากปัจจัยภายนอกหลายอย่าง เช่น สภาพอากาศ นโยบาย และตลาดโลก แนวทางแบบดั้งเดิมใช้โมเดลเศรษฐมิติ เช่น ARIMA ซึ่งเหมาะสำหรับข้อมูลที่มีความนิ่งและมีรูปแบบเชิงเส้น แต่ไม่สามารถจับความสัมพันธ์ที่ไม่เป็นเชิงเส้น GARCH ซึ่งจำลองความผันผวนที่เปลี่ยนแปลงตามเวลา ใช้สำหรับการจัดการความเสี่ยงและการกำหนดราคาออปชัน และ VAR ซึ่งจำลองหลาย Time Series พร้อมกันและจับความสัมพันธ์ข้ามสินค้า

แนวทางสมัยใหม่ใช้ Machine Learning โดยมีหลายอัลกอริทึม ได้แก่ Support Vector Regression ที่ใช้ kernel trick สำหรับ non-linear regression ตัวอย่างเช่น งานวิจัยของ Tian และคณะ (2015) ใช้ SVR สำหรับการพยากรณ์ราคาข้าวโพดได้ RMSE = 0.12 Random Forest Regression ที่เป็น Ensemble ของ Decision Trees สามารถจัดการกับความสัมพันธ์ที่ไม่เป็นเชิงเส้นและปฏิสัมพันธ์ ตัวอย่างเช่น งานวิจัยของ Ribeiro และ Oliveira (2011) ใช้ Random Forest สำหรับการทำนายราคาถั่วเหลือง Neural Networks โดยเฉพาะ LSTM ที่เหมาะสำหรับข้อมูลลำดับ ตัวอย่างเช่น งานวิจัยของ Xiong และคณะ (2018) ใช้ LSTM สำหรับสินค้าเกษตรได้ MAE = 2.3% และ Gradient Boosting (XGBoost, LightGBM) ที่เป็น state-of-the-art สำหรับข้อมูลแบบตาราง สามารถจัดการกับค่าที่หายไปและปฏิสัมพันธ์ของ features

งานวิจัยบางชิ้นพยายามบูรณาการปัจจัยภายนอก เช่น ข้อมูลสภาพอากาศ (อุณหภูมิ ปริมาณฝน) ที่ส่งผลต่ออุปทาน ตัวชี้วัดทางเศรษฐกิจ (GDP เงินเฟ้อ) ที่ส่งผลต่ออุปสงค์ และตัวแปรนโยบาย (เงินอุดหนุน ภาษี) ที่ส่งผลต่อตลาด ตัวอย่างเช่น งานวิจัยของ Ubilava (2012) ศึกษาผลกระทบของ El Niño ต่อราคาสินค้าเกษตร อย่างไรก็ตาม งานวิจัยส่วนใหญ่ยังมีปัญหา Temporal Bias สูง (พึ่งพาราคาในอดีตมากเกินไป) มีความแข็งแกร่งต่อการกระทบกระเทือนของตลาดจำกัด มีการบูรณาการข้อมูลสภาพอากาศและเศรษฐกิจไม่เพียงพอ และขาดวิธีการวัด Bias อย่างเป็นระบบ

### 2.3.4 ระบบสนับสนุนการตัดสินใจทางการเกษตร

ระบบสนับสนุนการตัดสินใจทางการเกษตรมีทั้งระบบเชิงพาณิชย์และต้นแบบการวิจัย ระบบเชิงพาณิชย์ที่สำคัญ ได้แก่ Climate FieldView ของ Bayer/Monsanto ที่ให้บริการภาพถ่ายดาวเทียม การทำแผนที่ฟาร์ม การวิเคราะห์ผลผลิต และคำแนะนำการปลูกและเก็บเกี่ยว แต่เน้นที่ฟาร์มขนาดใหญ่และตลาดสหรัฐอเมริกา John Deere Operations Center ที่ให้บริการข้อมูลจากเครื่องจักร การดำเนินงานในฟาร์ม และแผนที่สำหรับการใช้ปุ๋ยแบบแปรผัน แต่ต้องการอุปกรณ์ John Deere และมีราคาแพง Farmers Edge ที่ให้บริการการตรวจสอบสภาพอากาศ การพยากรณ์ การจำลองพืช และการทำนายผลผลิต และ Cropio ที่ให้บริการการตรวจสอบฟาร์มจากดาวเทียม การบูรณาการข้อมูลสภาพอากาศ และการจัดการงาน

ต้นแบบการวิจัยที่สำคัญ ได้แก่ DSSAT (Decision Support System for Agrotechnology Transfer) ที่มีโมเดลจำลองพืช (CERES, CROPGRO) สำหรับการประเมินผลกระทบของการเปลี่ยนแปลงสภาพภูมิอากาศ แต่ต้องการการปรับแต่งอย่างละเอียดและความเชี่ยวชาญ APSIM (Agricultural Production Systems sIMulator) ที่มีโมเดลพืชและดินแบบโมดูลาร์ สำหรับการวิเคราะห์ความยั่งยืนระยะยาว ใช้กันอย่างแพร่หลายในการวิจัย แต่ไม่เป็นมิตรกับเกษตรกร และ CropSyst ที่จำลองระบบการปลูกพืช การจัดการน้ำและไนโตรเจน และการวิเคราะห์สถานการณ์สภาพภูมิอากาศ

เมื่อเปรียบเทียบกับระบบที่มีอยู่ ระบบ FarmMe มีจุดเด่นหลายประการ ประการแรก เป็นระบบแบบครบวงจร (End-to-End) ที่ครอบคลุมตลอดวงจรการเพาะปลูก ตั้งแต่การวางแผนไปจนถึงการเก็บเกี่ยว ในขณะที่ระบบอื่นมักเน้นเพียงด้านใดด้านหนึ่ง ประการที่สอง ใช้ Multi-Objective Optimization ที่สมดุลระหว่างกำไร ความเสี่ยง และความยั่งยืน ในขณะที่ระบบอื่นมักเพิ่มประสิทธิภาพเพียงวัตถุประสงค์เดียว ประการที่สาม มีการป้องกัน Data Leakage อย่างเข้มงวด ซึ่งเป็นปัญหาที่พบบ่อยในงานวิจัย Agricultural ML แต่มักถูกมองข้าม ประการที่สี่ ออกแบบมาสำหรับเกษตรกรรายย่อย มีความต้องการโครงสร้างพื้นฐานต่ำ ในขณะที่ระบบเชิงพาณิชย์ส่วนใหญ่เน้นที่ฟาร์มขนาดใหญ่ ประการที่ห้า เฉพาะสำหรับประเทศไทย ครอบคลุม 77 จังหวัดและ 46 ชนิดพืช ในขณะที่ระบบอื่นมักเน้นที่สหรัฐอเมริกาหรือยุโรป

อย่างไรก็ตาม ระบบ FarmMe ยังมีข้อจำกัดเมื่อเทียบกับระบบเชิงพาณิชย์ ได้แก่ ใช้ข้อมูลสังเคราะห์ (ยังไม่ได้ตรวจสอบกับข้อมูลจริง) มีการบูรณาการเซ็นเซอร์จำกัด ไม่มีภาพถ่ายดาวเทียม และอยู่ในระดับการพิสูจน์แนวคิด ยังไม่พร้อมสำหรับการใช้งานขนาดใหญ่

## 2.4 สรุป

บทนี้ได้นำเสนอทฤษฎีและงานวิจัยที่เป็นพื้นฐานของระบบ FarmMe ครอบคลุมห้าสาขาหลัก ได้แก่ Machine Learning Fundamentals ที่อธิบายแนวคิดพื้นฐานของ Supervised Learning, Regression, Classification และตัววัดประสิทธิภาพ Multi-Objective Optimization และ NSGA-II ที่อธิบายแนวคิดของ Pareto Optimality และอัลกอริทึม NSGA-II สำหรับการหาคำตอบที่สมดุลระหว่างวัตถุประสงค์ที่ขัดแย้งกัน Gradient Boosting และ XGBoost ที่อธิบายเทคนิค Ensemble Learning และข้อดีของ XGBoost สำหรับข้อมูลแบบตาราง Time Series Forecasting ที่อธิบายวิธีการพยากรณ์ข้อมูลลำดับเวลา Feature Engineering และการป้องกัน Data Leakage และ Multi-Armed Bandits และ Thompson Sampling ที่อธิบายปัญหาการสมดุลระหว่างการสำรวจและการใช้ประโยชน์ และอัลกอริทึม Bayesian สำหรับการตัดสินใจลำดับ

นอกจากนี้ บทนี้ยังได้นำเสนอนิยามศัพท์เฉพาะที่ใช้ตลอดทั้งเอกสาร ครอบคลุมศัพท์ทางด้าน Machine Learning, การเกษตร และการเพิ่มประสิทธิภาพ เพื่อให้ผู้อ่านเข้าใจความหมายและบริบทของศัพท์เหล่านี้อย่างชัดเจน และได้ทบทวนงานวิจัยที่เกี่ยวข้อง ทั้งระบบเชิงพาณิชย์และต้นแบบการวิจัย เพื่อแสดงให้เห็นถึงบริบทและช่องว่างการวิจัยที่ระบบ FarmMe พยายามแก้ไข

ทฤษฎีและงานวิจัยที่นำเสนอในบทนี้จะถูกนำไปประยุกต์ใช้ในการพัฒนาระบบ FarmMe ตามที่จะอธิบายในบทที่ 3 โดยบทที่ 3 จะอธิบายขั้นตอนการดำเนินงานตั้งแต่การเก็บรวบรวมข้อมูลไปจนถึงการพัฒนาแอปพลิเคชัน และระบุว่าใช้ทฤษฎีใดจากบทนี้ในแต่ละขั้นตอน
