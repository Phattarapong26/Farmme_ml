# บทที่ 5: สรุปและอภิปรายผล

---

## 5.1 สรุปผลการทดลอง

[เนื้อหาจะถูกย้ายจาก Chapter10 และ Chapter11]

### 5.1.1 สรุปผลการพัฒนาระบบ

การวิจัยนี้ได้พัฒนาระบบสนับสนุนการตัดสินใจทางการเกษตรอัจฉริยะ (FarmMe) ที่ใช้เทคโนโลยี Machine Learning เพื่อช่วยเกษตรกรในการตัดสินใจตลอดวงจรการเกษตร ตั้งแต่การเลือกพืช การกำหนดเวลาปลูก การพยากรณ์ราคา ไปจนถึงการตัดสินใจเวลาเก็บเกี่ยว ระบบประกอบด้วย 4 โมเดลหลักที่ทำงานร่วมกันอย่างเป็นระบบ โดยแต่ละโมเดลได้รับการออกแบบและพัฒนาตามหลักการทางวิชาการและทฤษฎีที่เหมาะสม

ระบบใช้ข้อมูลสังเคราะห์ที่สร้างด้วย GPU-Accelerated Data Generation Pipeline ครอบคลุม 77 จังหวัด 46 ชนิดพืช และระยะเวลา 2 ปี รวมกว่า 2.3 ล้านระเบียดข้อมูล การสร้างข้อมูลคำนึงถึงความสัมพันธ์เชิงพื้นที่และเชิงเวลาอย่างเป็นระบบ เพื่อให้ข้อมูลมีความสมจริงและสะท้อนลักษณะของข้อมูลการเกษตรจริง การพัฒนาระบบเน้นการป้องกัน Data Leakage อย่างเข้มงวด โดยใช้เฉพาะข้อมูลที่มีอยู่ก่อนเวลาทำนาย และใช้ Temporal Split ในการแบ่งข้อมูลฝึกและทดสอบ

ผลการทดลองแสดงให้เห็นว่าระบบสามารถทำงานได้ตามวัตถุประสงค์ที่กำหนดไว้ โดยโมเดล NSGA-II + XGBoost สำหรับการแนะนำการเลือกพืชให้ผลลัพธ์ที่ดีที่สุดด้วย Test R² = 0.9944 ซึ่งแสดงให้เห็นว่าโมเดลสามารถอธิบายความแปรปรวนของข้อมูลได้ 99.44% และไม่มีปัญหา Overfitting ระบบได้รับการพัฒนาเป็นแอปพลิเคชันเว็บที่มี Backend API และ Frontend Dashboard เพื่อให้เกษตรกรสามารถใช้งานได้ง่าย

### 5.1.2 สรุปผลการทดลองแต่ละโมเดล

[ย่อหน้าสรุปผลแต่ละโมเดล]

**Model A (Crop Recommendation)**: โมเดล NSGA-II + XGBoost สามารถสร้าง Pareto Front ที่แสดง trade-offs ระหว่างกำไร ความเสี่ยง และความยั่งยืน ช่วยให้เกษตรกรเลือกพืชตามความต้องการของตนเอง

**Model B (Planting Window)**: โมเดล Logistic Regression with Cyclical Features สามารถจำแนกช่วงเวลาปลูกที่เหมาะสมได้ด้วย F1 Score = 0.70-0.75

**Model C (Price Forecasting)**: โมเดล XGBoost with Weather and Economic Features สามารถพยากรณ์ราคาได้ด้วย MAE = 13.31 บาท และลด Temporal Bias จาก 96.79% เหลือ 68.09%

**Model D (Harvest Timing)**: โมเดล Thompson Sampling สามารถตัดสินใจเวลาเก็บเกี่ยวได้ด้วยความแม่นยำ 68%

## 5.2 อภิปรายผล

[เนื้อหาจะถูกย้ายจาก Chapter10]

### 5.2.1 ทำไม NSGA-II + XGBoost จึงดีที่สุด

[ย่อหน้าอธิบายเหตุผลที่ NSGA-II + XGBoost ดีที่สุด]

โมเดล NSGA-II + XGBoost ให้ผลลัพธ์ที่ดีที่สุดเนื่องจากหลายเหตุผล ประการแรก NSGA-II เป็นอัลกอริทึมที่ออกแบบมาสำหรับการเพิ่มประสิทธิภาพหลายวัตถุประสงค์โดยเฉพาะ สามารถหา Pareto Front ได้อย่างมีประสิทธิภาพด้วย Fast Non-Dominated Sorting และ Crowding Distance ประการที่สอง XGBoost เป็นอัลกอริทึม Gradient Boosting ที่มีประสิทธิภาพสูงสำหรับข้อมูลแบบตาราง สามารถจัดการกับความสัมพันธ์ที่ซับซ้อนและ missing values ได้ดี ประการที่สาม การผสมผสานระหว่าง NSGA-II และ XGBoost ทำให้ได้ประโยชน์จากทั้งสองอัลกอริทึม โดย NSGA-II ทำหน้าที่ค้นหาโซลูชันที่หลากหลาย ขณะที่ XGBoost ทำหน้าที่ประเมินคุณภาพของแต่ละโซลูชันอย่างแม่นยำ

### 5.2.2 การอ้างอิงงานวิจัยที่สนับสนุน

[ย่อหน้าอ้างอิงงานวิจัยที่สนับสนุนผล]

ผลการทดลองนี้สอดคล้องกับงานวิจัยหลายชิ้นที่แสดงให้เห็นว่า NSGA-II เป็นอัลกอริทึมที่มีประสิทธิภาพสำหรับปัญหา Multi-Objective Optimization (Deb et al., 2002) และ XGBoost เป็นอัลกอริทึมที่ให้ผลลัพธ์ดีที่สุดสำหรับข้อมูลแบบตาราง (Chen & Guestrin, 2016) งานวิจัยของ Sarker & Ray (2009) ได้ใช้ Multi-Objective Optimization สำหรับการวางแผนการปลูกพืชในบังกลาเทศและพบว่าสามารถเพิ่มประสิทธิภาพได้ดีกว่าการใช้ Single-Objective Optimization นอกจากนี้ งานวิจัยของ Khosla et al. (2010) ได้ใช้ Gradient Boosting สำหรับการพยากรณ์ผลผลิตและพบว่าให้ผลลัพธ์ที่ดีกว่า Linear Regression และ Random Forest

### 5.2.3 การเปรียบเทียบกับงานอื่น

[ย่อหน้าเปรียบเทียบกับงานวิจัยอื่นที่คล้ายกัน]

เมื่อเปรียบเทียบกับงานวิจัยอื่นๆ ระบบ FarmMe มีข้อได้เปรียบหลายประการ ประการแรก ระบบนี้เป็นระบบแบบครบวงจรที่ครอบคลุมตั้งแต่การเลือกพืชไปจนถึงการเก็บเกี่ยว ในขณะที่ระบบอื่นๆ มักเน้นเพียงด้านใดด้านหนึ่ง ประการที่สอง ระบบนี้ใช้ Multi-Objective Optimization ซึ่งช่วยให้เกษตรกรสามารถเลือกพืชตามความต้องการของตนเอง ไม่ใช่แค่เพิ่มกำไรเพียงอย่างเดียว ประการที่สาม ระบบนี้มีการป้องกัน Data Leakage อย่างเข้มงวด ซึ่งเป็นปัญหาที่พบบ่อยในงานวิจัย Agricultural ML แต่มักถูกมองข้าม อย่างไรก็ตาม ระบบนี้ยังมีข้อจำกัดคือใช้ข้อมูลสังเคราะห์ ไม่ได้ทดสอบกับข้อมูลจริงและเกษตรกรจริง

### 5.2.4 การลด Temporal Bias

[ย่อหน้าอภิปรายการลด Temporal Bias]

การลด Temporal Bias จาก 96.79% เหลือ 68.09% ในโมเดลพยากรณ์ราคาเป็นผลสำเร็จที่สำคัญ แสดงให้เห็นว่าการบูรณาการข้อมูลสภาพอากาศและตัวชี้วัดทางเศรษฐกิจสามารถลดการพึ่งพาราคาในอดีตได้อย่างมีนัยสำคัญ อย่างไรก็ตาม Temporal Bias ที่เหลืออยู่ 68.09% ยังคงสูง แสดงให้เห็นว่าโมเดลยังคงพึ่งพาราคาในอดีตอยู่มาก ซึ่งอาจทำให้โมเดลทำงานได้ไม่ดีในช่วงที่ตลาดผันผวนหรือมีเหตุการณ์ไม่คาดคิด งานในอนาคตควรพัฒนาวิธีการลด Temporal Bias เพิ่มเติม เช่น การใช้ Causal Inference หรือการบูรณาการข้อมูลเพิ่มเติม

## 5.3 ข้อจำกัดของการวิจัย

[เนื้อหาจะถูกย้ายจาก Chapter10 Section 10.7]

### 5.3.1 ข้อจำกัดด้านข้อมูล (Synthetic Data Limitations)

[ย่อหน้าอธิบายข้อจำกัดด้านข้อมูล]

ข้อจำกัดหลักของการวิจัยนี้คือการใช้ข้อมูลสังเคราะห์ (Synthetic Data) แทนข้อมูลจริง แม้ว่าข้อมูลสังเคราะห์จะถูกสร้างขึ้นโดยใช้โมเดลทางสถิติที่สะท้อนรูปแบบและความสัมพันธ์ที่สมจริง และได้รับการตรวจสอบกับสถิติการเกษตรที่ทราบ แต่ข้อมูลสังเคราะห์ไม่สามารถจับความซับซ้อนและความไม่แน่นอนของโลกจริงได้ทั้งหมด เช่น เหตุการณ์ไม่คาดคิด (Black Swan Events), พฤติกรรมของเกษตรกร, และปัจจัยทางสังคมและการเมือง ดังนั้น ประสิทธิภาพของระบบในโลกจริงอาจแตกต่างจากผลการทดลอง การใช้งานจริงต้องการการบูรณาการข้อมูลจริงและการทดสอบกับเกษตรกรจริง

### 5.3.2 ข้อจำกัดด้านขอบเขต (Scope Limitations)

[ย่อหน่อยอธิบายข้อจำกัดด้านขอบเขต]

การวิจัยนี้มีขอบเขตที่จำกัดในหลายด้าน ประการแรก ระยะเวลาข้อมูล 2 ปีอาจไม่เพียงพอสำหรับการจับแนวโน้มระยะยาว เช่น การเปลี่ยนแปลงสภาพภูมิอากาศ ประการที่สอง ข้อมูลสภาพอากาศและเศรษฐกิจที่ใช้เป็นข้อมูลพื้นฐาน ไม่รวมปัจจัยอื่นๆ ที่อาจมีผลกระทบ เช่น ลม รังสีดวงอาทิตย์ ความชื้นในดิน พลวัตตลาดระดับจุลภาค และรายละเอียดห่วงโซ่อุปทาน ประการที่สาม ระบบเน้นที่ประเทศไทยเท่านั้น ผลลัพธ์อาจไม่สามารถนำไปใช้กับประเทศอื่นได้โดยตรง

### 5.3.3 ข้อจำกัดด้านเทคนิค (Technical Limitations)

[ย่อหน้าอธิบายข้อจำกัดด้านเทคนิค]

ระบบมีข้อจำกัดด้านเทคนิคหลายประการ ประการแรก ระบบใช้ Classical ML แทน Deep Learning เนื่องจากข้อมูลไม่เพียงพอและความกังวลด้านการอธิบาย แต่ Deep Learning อาจให้ผลลัพธ์ที่ดีกว่าหากมีข้อมูลเพียงพอ ประการที่สอง ระบบเป็น Proof of Concept ไม่ได้ออกแบบมาสำหรับการใช้งานขนาดใหญ่ ต้องการวิศวกรรมเพิ่มเติมสำหรับการใช้งานจริง ประการที่สาม ระบบไม่มีการบูรณาการกับเซ็นเซอร์ IoT หรือภาพถ่ายดาวเทียม ซึ่งอาจช่วยเพิ่มประสิทธิภาพได้

## 5.4 ข้อเสนอแนะ

[เนื้อหาใหม่]

### 5.4.1 ข้อเสนอแนะสำหรับการใช้งานจริง

[ย่อหน้าเสนอแนะการใช้งานจริง]

สำหรับการนำระบบไปใช้งานจริง มีข้อเสนอแนะดังนี้ ประการแรก ควรทดสอบระบบกับข้อมูลจริงและเกษตรกรจริงในพื้นที่นำร่อง เพื่อประเมินประสิทธิภาพและความเหมาะสมในโลกจริง ประการที่สอง ควรพัฒนาระบบให้รองรับการบูรณาการกับข้อมูลจริงจากหลายแหล่ง เช่น ข้อมูลสภาพอากาศจากกรมอุตุนิยมวิทยา ข้อมูลราคาจากกระทรวงพาณิชย์ และข้อมูลจากเซ็นเซอร์ IoT ประการที่สาม ควรพัฒนาระบบให้มีความแข็งแกร่งและพร้อมสำหรับการใช้งานขนาดใหญ่ รวมถึงการจัดการข้อผิดพลาด การสำรองข้อมูล และการรักษาความปลอดภัย

### 5.4.2 ข้อเสนอแนะสำหรับการพัฒนาต่อ

[ย่อหน้าเสนอแนะการพัฒนาต่อ]

สำหรับการพัฒนาระบบต่อไป มีข้อเสนอแนะดังนี้ ประการแรก ควรพัฒนา User Interface ให้ใช้งานง่ายและเหมาะสมกับเกษตรกรรายย่อย รวมถึงการสนับสนุนภาษาไทยและการแสดงผลบนมือถือ ประการที่สอง ควรพัฒนาระบบให้มีความสามารถในการอธิบาย (Explainability) มากขึ้น เพื่อให้เกษตรกรเข้าใจและไว้วางใจคำแนะนำ ประการที่สาม ควรพัฒนาระบบให้สามารถเรียนรู้จากข้อมูลใหม่และปรับปรุงตัวเองอย่างต่อเนื่อง (Continuous Learning)

## 5.5 งานวิจัยในอนาคต

[เนื้อหาจะถูกย้ายจาก Chapter10 Section 10.11 และ Chapter11 Section 11.6]

### 5.5.1 การทดสอบในโลกจริง (Real-World Validation)

[ย่อหน้าอธิบายงานวิจัยในอนาคต]

งานวิจัยในอนาคตที่สำคัญที่สุดคือการทดสอบระบบในโลกจริงกับเกษตรกรจริง ควรดำเนินการทดลองแบบ Field Trial ในพื้นที่นำร่อง โดยเปรียบเทียบผลลัพธ์ระหว่างเกษตรกรที่ใช้ระบบและไม่ใช้ระบบ (A/B Testing) และประเมินผลกระทบระยะยาวต่อรายได้ ความเสี่ยง และความยั่งยืน นอกจากนี้ ควรศึกษาพฤติกรรมการใช้งานของเกษตรกร ความพึงพอใจ และอุปสรรคในการนำไปใช้

### 5.5.2 การพัฒนาโมเดลขั้นสูง (Advanced Modeling)

[ย่อหน้าอธิบายงานวิจัยในอนาคต]

งานวิจัยในอนาคตควรพัฒนาโมเดลที่ซับซ้อนและแม่นยำมากขึ้น เช่น การใช้ Deep Learning สำหรับการพยากรณ์ราคาและผลผลิต การใช้ Causal Inference สำหรับการเข้าใจความสัมพันธ์เชิงสาเหตุ การใช้ Transfer Learning สำหรับการปรับโมเดลให้เหมาะกับภูมิภาคต่างๆ และการใช้ Multi-Agent Systems สำหรับการประสานการตัดสินใจระหว่างเกษตรกรหลายคน

### 5.5.3 การบูรณาการข้อมูลเพิ่มเติม (Additional Data Integration)

[ย่อหน้าอธิบายงานวิจัยในอนาคต]

งานวิจัยในอนาคตควรบูรณาการข้อมูลเพิ่มเติมเพื่อเพิ่มประสิทธิภาพของระบบ เช่น ข้อมูลจากเซ็นเซอร์ IoT (ความชื้นในดิน อุณหภูมิ ความชื้นอากาศ) ภาพถ่ายดาวเทียม (NDVI, Crop Health) ข้อมูลตลาดแบบเรียลไทม์ (ราคา อุปสงค์ อุปทาน) และข้อมูลทางสังคม (ความต้องการของผู้บริโภค เทรนด์อาหาร)

### 5.5.4 การปรับตัวต่อการเปลี่ยนแปลงสภาพภูมิอากาศ (Climate Change Adaptation)

[ย่อหน้าอธิบายงานวิจัยในอนาคต]

งานวิจัยในอนาคตควรพัฒนาระบบให้สามารถปรับตัวต่อการเปลี่ยนแปลงสภาพภูมิอากาศ เช่น การใช้ Climate Scenarios สำหรับการพยากรณ์ระยะยาว การพัฒนา Adaptive Decision-Making ที่สามารถปรับเปลี่ยนคำแนะนำตามสถานการณ์ และการเพิ่มประสิทธิภาพความยืดหยุ่น (Resilience Optimization) เพื่อลดความเสี่ยงจากเหตุการณ์สภาพอากาศรุนแรง

## 5.6 สรุป

[ย่อหน้าสรุปทั้งเล่ม]

การวิจัยนี้ได้พัฒนาระบบสนับสนุนการตัดสินใจทางการเกษตรอัจฉริยะ (FarmMe) ที่ใช้เทคโนโลยี Machine Learning โดยเน้นที่ระบบแนะนำการเลือกพืช ผลการทดลองแสดงให้เห็นว่าโมเดล NSGA-II + XGBoost สามารถแนะนำพืชได้อย่างมีประสิทธิภาพด้วย R² = 0.47 และสร้าง Pareto Front ที่แสดง trade-offs ระหว่างกำไร ความเสี่ยง และความยั่งยืน ระบบนี้มีข้อได้เปรียบคือครอบคลุมตลอดวงจรการเกษตร ใช้ Multi-Objective Optimization และมีการป้องกัน Data Leakage อย่างเข้มงวด อย่างไรก็ตาม ระบบยังมีข้อจำกัดคือใช้ข้อมูลสังเคราะห์และยังไม่ได้ทดสอบในโลกจริง งานวิจัยในอนาคตควรเน้นการทดสอบในโลกจริง การพัฒนาโมเดลขั้นสูง การบูรณาการข้อมูลเพิ่มเติม และการปรับตัวต่อการเปลี่ยนแปลงสภาพภูมิอากาศ
