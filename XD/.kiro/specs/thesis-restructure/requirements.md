# Requirements Document

## Introduction

โครงการนี้มีวัตถุประสงค์เพื่อปรับโครงสร้างเอกสารวิชาการ FarmMe ให้สอดคล้องกับ feedback จากอาจารย์ที่ปรึกษา โดยมีการเปลี่ยนแปลงโครงสร้างหลักของเอกสารจากรูปแบบเดิมที่เน้นแยกตามโมเดล ไปเป็นรูปแบบใหม่ที่เน้นการดำเนินงานตามขั้นตอนวิจัย

## Glossary

- **System**: ระบบเอกสารวิชาการ FarmMe ที่ประกอบด้วยบทต่างๆ
- **Chapter**: บทในเอกสารวิชาการ
- **Feedback**: ข้อเสนอแนะจากอาจารย์ที่ปรึกษา
- **Restructure**: การปรับโครงสร้างเอกสารใหม่
- **Temporal Bias**: ความลำเอียงทางเวลาในโมเดล ML
- **Data Leakage**: การรั่วไหลของข้อมูลในโมเดล ML
- **Crop Recommendation**: ระบบแนะนำการเลือกพืช

## Requirements

### Requirement 1: ปรับข้อมูลหน้าปกและอาจารย์

**User Story:** ในฐานะผู้เขียนเอกสาร ฉันต้องการเพิ่มข้อมูลอาจารย์ที่ปรึกษาและวิชาที่เกี่ยวข้องให้ครบถ้วน เพื่อให้เอกสารมีความสมบูรณ์ตามมาตรฐานวิชาการ

#### Acceptance Criteria

1. WHEN ผู้ใช้เปิดเอกสาร THE System SHALL แสดงชื่ออาจารย์ที่ปรึกษาครบทั้ง 3 คน
2. WHEN ผู้ใช้ตรวจสอบข้อมูลวิชา THE System SHALL แสดงรายชื่อวิชาที่เกี่ยวข้องครบทั้ง 3 วิชา
3. THE System SHALL จัดรูปแบบข้อมูลอาจารย์และวิชาให้เป็นมาตรฐานวิชาการ

### Requirement 2: ปรับโครงสร้างบทที่ 2 เป็นทฤษฎีและงานวิจัยที่เกี่ยวข้อง

**User Story:** ในฐานะผู้อ่านเอกสาร ฉันต้องการให้บทที่ 2 นำเสนอเฉพาะทฤษฎีและงานวิจัยที่เกี่ยวข้อง โดยไม่รวมการอธิบายการนำไปใช้ เพื่อให้เข้าใจพื้นฐานทางทฤษฎีก่อนเข้าสู่การดำเนินงาน

#### Acceptance Criteria

1. THE System SHALL แยกบทที่ 2 เป็น 2 ส่วนหลัก คือ ทฤษฎีที่เกี่ยวข้อง และงานวิจัยที่เกี่ยวข้อง
2. THE System SHALL ลบบทนำออกจากบทที่ 2
3. WHEN ผู้อ่านอ่านบทที่ 2 THE System SHALL นำเสนอเฉพาะทฤษฎีที่ใช้จริงในโครงการเท่านั้น
4. THE System SHALL เขียนเนื้อหาในรูปแบบย่อหน้า (paragraph) แทนการใช้ bullet points
5. THE System SHALL รวมนิยามศัพท์เฉพาะไว้ในบทที่ 2 โดยขยายความให้ยาวกว่าเดิม
6. THE System SHALL ไม่รวมการอธิบายการนำทฤษฎีไปใช้ในบทที่ 2

### Requirement 3: สร้างบทที่ 3 ใหม่เป็นการดำเนินงาน

**User Story:** ในฐานะผู้อ่านเอกสาร ฉันต้องการให้บทที่ 3 อธิบายขั้นตอนการดำเนินงานตามลำดับ เพื่อให้เข้าใจกระบวนการวิจัยอย่างเป็นระบบ

#### Acceptance Criteria

1. THE System SHALL สร้างบทที่ 3 ใหม่ที่อธิบายการดำเนินงานตามลำดับ: Data Collection → EDA → Feature Engineering → Feature Selection → Model Training → Error Analysis
2. WHEN ผู้อ่านอ่านบทที่ 3 THE System SHALL อธิบายว่าใช้ทฤษฎีใดจากบทที่ 2 ในแต่ละขั้นตอน
3. THE System SHALL อธิบายการทำ one-hot encoding และเทคนิคอื่นๆ กับข้อมูลชุดใด
4. THE System SHALL ระบุแหล่งที่มาของข้อมูล
5. THE System SHALL อธิบายวิธีการเตรียมข้อมูล
6. THE System SHALL ระบุโมเดลที่ใช้และเหตุผลในการเลือก
7. THE System SHALL อธิบายวิธีการแบ่งข้อมูล (train/test split ratio)
8. THE System SHALL ระบุตัววัดประสิทธิภาพที่ใช้
9. THE System SHALL อธิบายวิธีการพัฒนาแอปพลิเคชัน
10. THE System SHALL ย้ายเนื้อหาที่เกี่ยวกับการนำไปใช้จากบทที่ 2 มาไว้ในบทที่ 3

### Requirement 4: ปรับบทที่ 4 เป็นผลการทดลอง

**User Story:** ในฐานะผู้อ่านเอกสาร ฉันต้องการให้บทที่ 4 นำเสนอผลการทดลองและการเปรียบเทียบโมเดล เพื่อให้เห็นประสิทธิภาพของระบบ

#### Acceptance Criteria

1. THE System SHALL สรุปประสิทธิภาพของแต่ละโมเดลในรูปแบบตาราง
2. THE System SHALL ระบุโมเดลที่ดีที่สุดพร้อมเหตุผล
3. IF มีแดชบอร์ด THEN THE System SHALL แสดงภาพหน้าจอแดชบอร์ดพร้อมคำอธิบาย
4. THE System SHALL สรุปผลตามวัตถุประสงค์ที่ระบุในบทที่ 1
5. THE System SHALL สรุปผลตามขอบเขตที่กำหนดในบทที่ 1
6. THE System SHALL เน้นที่โมเดล Crop Recommendation เป็นหลัก แต่ถ้ามีเวลาให้รวมทั้ง 4 โมเดล

### Requirement 5: ปรับบทที่ 5 เป็นสรุปและอภิปรายผล

**User Story:** ในฐานะผู้อ่านเอกสาร ฉันต้องการให้บทที่ 5 สรุปผลการทดลองและอภิปรายผลอย่างลึกซึ้ง เพื่อให้เข้าใจความหมายและข้อจำกัดของผลการวิจัย

#### Acceptance Criteria

1. THE System SHALL สรุปผลการทดลองทั้งหมดในบทที่ 5
2. THE System SHALL อภิปรายว่าทำไมโมเดลที่เลือกจึงดีที่สุด
3. THE System SHALL อ้างอิงงานวิจัยที่เกี่ยวข้องเพื่อสนับสนุนผลที่ได้
4. THE System SHALL ระบุข้อจำกัดของการวิจัย
5. THE System SHALL เสนอแนวทางการวิจัยในอนาคต
6. THE System SHALL รวมบรรณานุกรมที่สมบูรณ์

### Requirement 6: ปรับภาษาให้เป็นภาษาเขียนทางวิชาการ

**User Story:** ในฐานะผู้อ่านเอกสารวิชาการ ฉันต้องการให้เอกสารใช้ภาษาเขียนที่เป็นทางการ เพื่อให้เหมาะสมกับมาตรฐานวิชาการ

#### Acceptance Criteria

1. THE System SHALL แปลงภาษาพูดเป็นภาษาเขียนทางวิชาการ
2. THE System SHALL หลีกเลี่ยงคำถามในรูปแบบภาษาพูด เช่น "ปลูกเมื่อไหร่" "เลือกอะไร"
3. THE System SHALL ใช้คำศัพท์ทางวิชาการที่เหมาะสม
4. THE System SHALL รักษาความเป็นทางการตลอดทั้งเอกสาร

### Requirement 7: ปรับประโยชน์ที่คาดว่าจะได้รับให้สมจริง

**User Story:** ในฐานะผู้อ่านเอกสาร ฉันต้องการให้ประโยชน์ที่ระบุเป็นสิ่งที่วัดได้จริง เพื่อให้เอกสารมีความน่าเชื่อถือ

#### Acceptance Criteria

1. THE System SHALL ลบประโยชน์ที่วัดไม่ได้ออก เช่น "เพิ่มรายได้"
2. THE System SHALL ระบุเฉพาะประโยชน์ของแอปพลิเคชันที่วัดได้จริง
3. THE System SHALL เน้นประโยชน์ในระยะที่ทำได้จริงในปัจจุบัน
4. THE System SHALL หลีกเลี่ยงการอ้างถึงผลกระทบที่ไม่สามารถพิสูจน์ได้

### Requirement 8: ปรับรูปแบบการนำเสนอให้เป็นย่อหน้า

**User Story:** ในฐานะผู้อ่านเอกสาร ฉันต้องการให้เนื้อหาเขียนในรูปแบบย่อหน้าที่ต่อเนื่อง เพื่อให้อ่านง่ายและเป็นธรรมชาติ

#### Acceptance Criteria

1. THE System SHALL แปลง bullet points เป็นย่อหน้าในส่วนที่เหมาะสม
2. THE System SHALL รักษา bullet points เฉพาะในส่วนที่จำเป็น เช่น รายการขั้นตอน
3. THE System SHALL เขียนเนื้อหาให้ต่อเนื่องและเชื่อมโยงกัน
4. THE System SHALL หลีกเลี่ยงการใช้ bullet points สั้นๆ ที่ดูไม่เป็นทางการ

### Requirement 9: จัดการโครงสร้างบทให้สอดคล้องกัน

**User Story:** ในฐานะผู้อ่านเอกสาร ฉันต้องการให้โครงสร้างทั้งเล่มมีความสอดคล้องและเป็นระบบ เพื่อให้ติดตามเนื้อหาได้ง่าย

#### Acceptance Criteria

1. THE System SHALL จัดโครงสร้างเป็น: บทที่ 1 (บทนำ) → บทที่ 2 (ทฤษฎี) → บทที่ 3 (การดำเนินงาน) → บทที่ 4 (ผลการทดลอง) → บทที่ 5 (สรุปและอภิปราย)
2. THE System SHALL ตรวจสอบว่าแต่ละบทอ้างอิงบทก่อนหน้าได้อย่างถูกต้อง
3. THE System SHALL ตรวจสอบว่าวัตถุประสงค์ในบทที่ 1 สอดคล้องกับผลในบทที่ 4
4. THE System SHALL ตรวจสอบว่าทฤษฎีในบทที่ 2 ถูกนำไปใช้ในบทที่ 3

### Requirement 10: เพิ่มบรรณานุกรมที่สมบูรณ์

**User Story:** ในฐานะผู้อ่านเอกสารวิชาการ ฉันต้องการให้มีบรรณานุกรมที่สมบูรณ์ เพื่อให้สามารถตรวจสอบแหล่งอ้างอิงได้

#### Acceptance Criteria

1. THE System SHALL รวบรวมเอกสารอ้างอิงทั้งหมดที่ใช้ในเอกสาร
2. THE System SHALL จัดรูปแบบบรรณานุกรมตามมาตรฐาน APA หรือ IEEE
3. THE System SHALL ตรวจสอบว่าทุกการอ้างอิงในเนื้อหามีในบรรณานุกรม
4. THE System SHALL เรียงลำดับบรรณานุกรมตามตัวอักษร
